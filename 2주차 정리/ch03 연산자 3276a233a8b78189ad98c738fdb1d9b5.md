## 부호/증감 연산자

![image.png](ch03%20%EC%97%B0%EC%82%B0%EC%9E%90/image.png)

부호/증감 연산자는 변수의 값을 1씩 증가(++)시키거나 감소(-)시키는 단항 연산자를 말한다. + 연산자는 잘 사용 안 하지만 - 연산자는 변수값의 부호를 변경할 때 사용된다. 

```c
byte b = 100;
byte result = -b; //컴파일 에러
int result = -b;
```

byte의 허용 범위는 -128 ~ 127까지 여서 범위 안에 있는 거 같지만 앞에서 배운 연산시 자동 형 변환이라는 특성 때문에 byte 타입은 int 타입으로 바뀐다. 그래서 int 타입이 됐지만 byte result 때문에 int는 byte로 자동으로 바뀌지 않아서 컴파일 에러가 발생한다.

![image.png](ch03%20%EC%97%B0%EC%82%B0%EC%9E%90/image%201.png)

### 증감 연산자(Increment/Decrement Operators)

- 변수 값을 1씩 증가시키거나 감소시킨다.

증가 연산자(++):  i++ or ++i (1 증가)

감소 연산자(—): i — or — i (1 감소)

### 부호 연산자

- 숫자의 부호(양수/음수)를 변경하는 데 사용됩니다.
- 양수 부호(+): +x (값이 양수면 그대로, 음수면 양수로)
- 음수 부호(-): -x (값이 양수면 음수로, 양수면 음수로)

```c
int x = 1;
int y = 1;
int result1 = ++x + 10; // x를 1 증가 12
int result2 = y++ + 10; // y를 1 증가 12
```

++x 는 x가 먼저 증가해서 x = 2가 된다. 증가 먼저, 사용 나중

y++ 은 현재의 y의 값(1)을 사용하고 식이 끝난 뒤 y 증가한다. 사용 먼저, 증가 나중

즉 단독으로 쓰이면 결과 차이가 없지만 다른 여러 개의 연산자가 있으면 값이 달라질 수도 있다는 것이다.

```c
package operator;

public class IncreaseDecreaseOperatorExample {
    public static void main(String[] args) {

        int x = 10;
        int y = 10;
        int z;

        x++;
        ++x;

        System.out.println("x = " + x);

        System.out.println("-----------------------");

        y--;
        --y;

        System.out.println("y = " + y);

        System.out.println("-----------------------");

        z = x++;

        System.out.println("z = " + z);
        System.out.println("x = " + x);

        System.out.println("-----------------------");

        z = ++x;

        System.out.println("z = " + z);
        System.out.println("x = " + x);

        System.out.println("-----------------------");
        z = ++x + y++;

        System.out.println("z = " + z);
        System.out.println("x = " + x);
        System.out.println("y = " + y);

    }
}

```

## 산술 연산자

- 덧셈(+)

```c
int a = 5 + 3; // 8
```

- 뺄셈(-)

```c
int b = 5 - 3; // 2
```

- 곱셈(*)

```c
int c = 5 * 3; // 15
```

- 나눗셈(/)

```c
double d = (double) 5 / 3; // 1.6666666666666667
```

- 나머지(%)

```c
double f = 5.5 % 3; // 2.5
```

## 오버플로우와 언더플로우

- 오버플로우
    
    타입이 허용하는 최대값을 넘어갈 때 발생한다.
    
- 언더플로우
    
    타입이 허용하는 최소값을 더 작아질 때 발생한다.
    

즉 범위를 벗어났다는 것이다. 하지만 그렇다고 실행 에러가 발생하진 않는다. 해당 정수 타입의 최소값 또는 최대값으로 되돌아간다.

![image.png](ch03%20%EC%97%B0%EC%82%B0%EC%9E%90/image%205.png)

```c
byte value = 127;
value++;                     // value 값에 1을 더함
System.out.println(value);   // -128
value++;                     // value 값에 1을 더함
System.out.println(value);   // -127
```

여기서 byte 타입의 범위(-128 ~ 127)를 넘어갔기 때문에 오버플로우가 발생해서 최소값인 -128이 된다. 여기다가 다시 1을 더하면 -127이 된다. 

```c
byte value = -128;           
value--;                     // value 값에 1을 뺌
System.out.println(value);   // 127
value--;                     // value 값에 1을 뺌
System.out.println(value);   // 126
```

여기서도 마찬가지로 -128에서 1을 빼면 오버플로우가 발생해서 최대값인 127이 된다. 다시 1을 빼면 126이 된다.

byte뿐만 아니라 다른 타입들도 범위만 다를 뿐 오버플로우 및 언더플로우가 발생했을 때 최소값 또는 최대값으로 되돌아간다.

```c
package operator;

public class OverflowUnderflowExample {
    public static void main(String[] args) {

        byte var1 = 125;
        
        for(int i = 0; i < 5; i++) 
        {
            var1++;
            System.out.println("var1: " + var1);
        }
        
        System.out.println("-----------------------");
        
        byte var2 = -125;
        
        for(int i = 0; i < 5; i++) 
        {
            var2--;
            System.out.println("var2: " + var2);
        }
    }
}

```

### 주의사항

마지막으로 연산 과정 중에 발생하는 오버플로우 또는 언더플로우가 발생했을 경우 우리가 원하는 값이 아니기 때문에 미리 신경써서 더 큰 타입으로 연산을 하는 것이 좋다. 

```c
int x = 100000000;
int y = 100000000;
int z = x * y        // 1874919424 우리가 원하는 값이 아님

long x = 100000000;
long y = 100000000;
long z = x * y      // 10000000000000000
```

## 정확한 계산은 정수 연산으로

이 말은 우리가 컴퓨터 프로그래밍이나 금융 계산 등에서 부동 소수점(float, double) 연산의 미세한 오차를 피하고 싶을 때, 소수 부분을 제거한 정수 연산으로 변환하여 계산한 뒤 다시 소수점으로 변환하는 방법을 말한다.

즉 간단하게 말하자면 정확하게 계산하고 싶다면 실수 타입으로 하지 않는 것이 좋다는 말이다.

```c
int apple = 1;
double pieceUnit = 0.1;
int number = 7;

double result = apple - number*pieceUnit;

System.out.println("사과 1개에서 남은 양: " + result);
```

![image.png](ch03%20%EC%97%B0%EC%82%B0%EC%9E%90/image%207.png)

위처럼 코드를 짜면 result 값이 0.3이 되지 않는다. 이것은 부동 소수점 방식을 사용하는 실수 타입에서 자주 일어난다. 

```c
int apple = 1;
int totalPieces = apple * 10;
int number = 7;

int result = totalPieces - number;

System.out.println("10조각에서 남은 조각: " + result);
System.out.println("사과 1개에서 남은 양: " + result/10.0);
```

![image.png](ch03%20%EC%97%B0%EC%82%B0%EC%9E%90/image%208.png)

이렇게 변수를 정수로 하고 나중에 바꿔주는 방식으로 하는 것이 정확한 계산에는 좋다.

## 나눗셈 연산 후 NaN과 Infinity 처리

나눗셈 또는 나머지 연산 후 발생하는 Nan(Not a Number)과 Infinity(무한대)는 0으로 나누거나 실수 연산 시 예외 (ArithmeticException)가 발생한다. 무한대의 값을 정수로 표현할 수 없기 때문에 발생한다.

```c
int x = 5;
int y = 0;
int result = x / y;    //예외 발생
```

![image.png](ch03%20%EC%97%B0%EC%82%B0%EC%9E%90/image%209.png)

하지만 좌측 피연산자가 실수이거나 우측 피연산자가 0.0 또는 0.0f이면 예외가 발생하지 않고 연산의 결과는 Infinity 또는 NaN이 된다.

```c
int x = 5;
double y = 0.0;
double result = x %(/) y;   // % (Nan) / (Infinity) 
```

이 때 이 상태에서 계속해서 연산을 수행하면 안된다. 어떤 연산을 해도 NaN과 Infinity가 되므로 데이터가 엉망이 될 수 있다.

### 해결 방법

먼저 NaN과 Infinity가 되는지 확인을 해야 한다. 확인하기 위해서는 Double.isInfinite()와 Double.isNaN()을 사용한다.  이러면 맞으면 true 아니면 false라고 알려준다.

```c
package operator;

public class InfinityAndNaNCheckExample {
    public static void main(String[] args) {

        int x = 5;
        double y = 0.0;
        double z = x / y;   //double z = x % y; 둘 다 잘못된 코드이다.

        System.out.println(z + 2);

        if(Double.isInfinite(z) || Double.isNaN(z)) 
        {
            System.out.println("값 산출 불가");
        } 
        else 
        {
            System.out.println(z + 2);
        }
    }
}

```

if를 이용해서 NaN 또는 Infinity이면 산출 불가

아니면 연산 실행이다.

## 비교 연산자

두 피연산자의 크기나 동일성을 비교하여 참(true)또는 거짓(false)을 반환하는 연산자를 말한다. 비교 연산자는 주로 흐름 제어문인 조건문(if), 반복문(for, while)에서 실행 흐름을 제어할 때 주로 사용된다.

- ==(같음)
    
    두 값의 값이 같은지 비교한다.
    
    ```c
    1 == 1 (true)
    ```
    
- ≠(같지 않음)
    
    두 값의 값이 다른지 비교한다.
    
    ```c
    1 != 2 (true)
    ```
    
- <(작음)
    
    왼쪽 값이 오른쪽 값보다 작은지 비교한다.
    
    ```c
    1 < 2 (true)
    ```
    
- >(큼)
    
    왼쪽 값이 오른쪽 값보다 큰지 비교한다.
    
    ```c
    2 > 1 (true)
    ```
    
- ≤(작거나 같음)
    
    왼쪽 값이 오른쪽 값보다 작거나 같은지 비교한다.
    
    ```c
    1 <= 1 (true)
    ```
    
- ≥(크거나 같음)
    
    왼쪽 값이 오른쪽 값보다 크거나 같은지 비교한다.
    
    ```c
    2 >= 1(true)
    ```
    

![image.png](ch03%20%EC%97%B0%EC%82%B0%EC%9E%90/image%2010.png)

여기서 한 가지 예외가 있다. 0.1f == 0.1에서 double 타입으로 변환되면 0.1 == 0.1이 되어 true가 산출되어야 하지만, 이 결과값은 false가 산출된다.

```c
0.1f == 0.1 // false
```

실수 타입은 전에 말했던 것처럼 0.1을 정확히 표현할 수 없을 뿐만 아니라 float 타입과 double 타입의 정밀도 차이 때문이다. 이럴 때는 강제 타입 변환을 시키면 된다.

```c
0.1f == (float) 0.1 // true
```

문자열을 비교할 때에는 동등(==, ≠) 연산자 대신 equals()와 !equals()를 사용한다. 

```c
boolean result = str1.equals(str2);  //문자열이 같은지 검사(대소문자 구분)
               원본 문자열 비교 문자열
boolean result = ! str1.equals(str2); //문자열이 다른지 검사
```

```c
package operator;

public class CompareOperatorExample {
    public static void main(String[] args) {

        int num1 = 10;

        String str2 = "Java";boolean result8 = (str1.equals(str2));

    }
}

```

## 논리 연산자

참(True)과 거짓(False)을 다루며, 두 개 이상의 논리식이나 값을 연결하여 하나의 참/거짓 값을 만드는 연산자를 말한다.

주로 흐름 제어문인 조건문(if), 반복문(for, while) 등에서 주로 사용한다.

![image.png](ch03%20%EC%97%B0%EC%82%B0%EC%9E%90/image%2012.png)

&&와 &는 산출 결과는 같지만 연산 과정이 조금 다르다. 

&&는 앞의 피연산자가 false라면 뒤의 피연산자를 평가하지 않고 바로 false를 산출한다. 하지만 &는 두 피연산자 모두를 평가해서 산출 결과를 낸다. 그러므로 &보다 &&가 더 효율적으로 동작한다.

||이랑 |도 마찬가지이다. 

||는 앞의 피연산자가 true라면 뒤의 피연산자를 평가하지 않고 바로 true를 산출하지만, |는 두 피연산자 모두를 평가해서 산출 결과를 낸다.

```c
package operator;

public class LogicalOperatorExample {
    public static void main(String[] args) {

        int charCode = 'A';//int charCode = 'a';  //int charCode = '5';
        
        if ( (65 <= charCode) & (charCode <= 90) )
        {
            System.out.println("대문자이군요.");
        }
        
        if ( (97 <= charCode) && (charCode <= 122) )
        {
            System.out.println("소문자이군요.");
        }

        if ( (48 <= charCode) && (charCode <= 57) )
        {
            System.out.println("0~9 숫자이군요.");
        }

        int value = 6;   //int value = 7;

        if ( (value % 2 == 0) | (value % 3 == 0) )
        {
            System.out.println("2 또는 3의 배수이군요.");
        }
        
        boolean result = (value % 2 == 0) || (value % 3 == 0);
        
        if ( !result ) 
        {
            System.out.println("2 또는 3의 배수가 아니군요.");
        }
    }
    
} 
    

```

## 비트 논리 연산자

---

비트 논리 연산자는 byte, short, char 타입 피연산자를 int 타입으로 자동 변환한 후 연산을 수행한다. 

```c
byte num1 = 45;
byte num2 = 25;
byte result = num1 & num2;   //컴파일 에러
int result = num1 & num2;

System.out.println(result);
```

![image.png](ch03%20%EC%97%B0%EC%82%B0%EC%9E%90/image%2019.png)

byte 타입에서 int 타입으로 자동 변환되었는데 int가 아닌 byte 변수에 대입하려고 해서 컴파일 에러가 났다.

### 비트 논리 연산이 필요한 이유

둘 다 8비트이지만 허용 범위가 다르고 Java의 경우 음수까지도 포함하기 때문에 같은 값을 넣어도 결과가 다르다.

즉 같은 8비트 숫자라도 언어마다 해석이 다르기 때문에 비트 그대로 다루려면 비트 논리 연산이 필요하다.

![image.png](ch03%20%EC%97%B0%EC%82%B0%EC%9E%90/image%2020.png)

![image.png](ch03%20%EC%97%B0%EC%82%B0%EC%9E%90/image%2021.png)

```java
byte receiveData = -120; 
int unsignedInt = receiveData & 255;   //136
```

또 자바는 개발자의 편리성을 위해 Byte.toUnsignedInt() 코드가 있다.

```java
byte receiveData = -120; 
int unsignedInt = Byte.toUnsignedInt(receiveData);   //136
```

```java
package operator;

public class BitLogicExample {
    public static void main(String[] args) {

        System.out.println("45 & 25 = " + (45 & 25));
        System.out.println("45 | 25 = " + (45 | 25));
        System.out.println("45 ^ 25 = " + (45 ^ 25));
        System.out.println("~45 = " + (~45));

        System.out.println("-------------------------------");

        byte receiveData = -128;

        //방법1: 비트 논리곱 연산으로 Unsigned 정수 얻기
        int unsignedInt1 =  receiveData & 255;

        System.out.println(unsignedInt1);

        //방법2: 자바 API를 이용해서 Unsigned 정수 얻기

        int unsignedInt2 = Byte.toUnsignedInt(receiveData);

        System.out.println(unsignedInt2);

        int test = 136;
        byte btest = (byte) test;

        System.out.println(btest);

    }
}

```

## 비트 이동 연산자

정수형 데이터의 이진수 비트들을 왼쪽 또는 오른쪽으로 지정된 수만큼 이동시키는 연산자를 말한다.

![image.png](ch03%20%EC%97%B0%EC%82%B0%EC%9E%90/image%2023.png)

좌측 이동 연산자를 사용하여 왼쪽으로 이동 시켜보았다.

```java
int result = 1 << 3;
```

왼쪽으로 3비트 이동하면 이동하고 빈 공간을 0으로 채운다.

![image.png](ch03%20%EC%97%B0%EC%82%B0%EC%9E%90/image%2024.png)

0001(1)에서 1000(8)으로 바뀐 모습은 마치 2의 거듭제곱을 곱한 결과와 동일한다. 

```java
1 << 3  =  1 x 23  =  8
```

이번에는 우측 이동 연산자를 사용하여 오른쪽으로 이동 시켜보았다

```java
int result = -8 >> 3;
```

우측으로 이동하면 오른쪽 3비트는 밀려서 버려지고, 맨 왼쪽에 새로 생기는 3비트는 최상위 부호 비트와 동일한 값으로 채워진다.

만약 4byte가 모두 1이면 정수 -1이므로 -1이 저장 된다.

![image.png](ch03%20%EC%97%B0%EC%82%B0%EC%9E%90/image%2025.png)

-8에서 3으로 바뀐 모습은 마치 2의 거듭제곱을 나눈 결과와 동일하다.

```java
-8 >> 3 = -8 / 2^3 = -1
```

```java
package operator;

public class BitShiftExample1 {
    public static void main(String[] args) {

        int num1 = 1;
        int result1 = num1 << 3;
        int result2 = num1 * (int) Math.pow(2, 3);

        System.out.println("result1: " + result1);
        System.out.println("result2: " + result2);

        int num2 = -8;
        int result3 = num2 >> 3;
        int result4 = num2 / (int) Math.pow(2, 3);

        System.out.println("result3: " + result3);
        System.out.println("result4: " + result4);

    }
}

```

![image.png](ch03%20%EC%97%B0%EC%82%B0%EC%9E%90/image%2026.png)

이번에는 우측 이동 연산자(>>>)를 사용해서 오른쪽으로 이동 시켜보았다.

```java
int result = -8 >>> 3;
```

32비트 전체를 오른쪽으로 3비트 이동하면 >> 와 같이 오른쪽 3비트는 밀려서 버려지지만 맨 왼쪽에 새로 생기는 비트는 무조건 0으로 채워진다는게 >> 와 차이점이다.

![image.png](ch03%20%EC%97%B0%EC%82%B0%EC%9E%90/image%2027.png)

```java
package operator;

public class BitShiftExample2 {
    public static void main(String[] args) {
        int value = 772;  //[00000000] [00000000] [00000011] [00000100]

        //우측으로 3byte(24bit) 이동하고 끝 1바이트만 읽음: [00000000]

        byte byte1 = (byte) (value >>> 24);
        int int1 = byte1 & 255;

        System.out.println("첫 번째 바이트 부호 없는 값: " +  int1);

        //우측으로 2byte(16bit) 이동하고 끝 1바이트만 읽음: [00000000]

        byte byte2 = (byte) (value >>> 16);
        int int2 = Byte.toUnsignedInt(byte2);

        System.out.println("두 번째 바이트 부호 없는 값: " +  int2);

        //우측으로 1byte(8bit) 이동하고 끝 1바이트만 읽음: [00000011]

        byte byte3 = (byte) (value >>> 8);
        int int3 = byte3 & 255;
        System.out.println("세 번째 바이트 부호 없는 값: " +  int3);

        //끝 1바이트만 읽음: [00000100]

        byte byte4 = (byte) value;
        int int4 = Byte.toUnsignedInt(byte4);

        System.out.println("네 번째 바이트 부호 없는 값: " +  int4);

        
    }
}

```

## 대입 연산자

값을 변수에 넣는 연산자라고 보면 된다.

- 보통 대입 연산자 왼쪽에는 “값을 저장 할 수 있는 대상”인 변수가 온다.
- 대입 연산자 오른쪽인 우측 피연산자는 리터럴 및 변수 그리고 다른 연산식이 올 수 있다.

### 대입 연산자 종류

- 단순 대입 연산자
    
    → 단순히 값을 대입하는 연산자
    
- 복합 대입 연산자
    
    → 정해진 연산을 수행한 후 결과를 대입하는 복합 대입 연산자
    
    쓰는 이유
    
    - 코드가 짧아짐
    - 실수 줄어듦
    - 자동 형 변환이 더 관대함

![image.png](ch03%20%EC%97%B0%EC%82%B0%EC%9E%90/image%2029.png)

```java
package operator;

public class AssignmentOperatorExample {
    public static void main(String[] args) {

        int result = 0;
        result += 10;

        System.out.println("result=" + result);

        result -= 5;

        System.out.println("result=" + result);

        result *= 3;

        System.out.println("result=" + result);

        result /= 5;

        System.out.println("result=" + result);

        result %= 3;

        System.out.println("result=" + result);

    }
}

```

## 삼향(조건) 연산자

> 세 개의 피연산자(operand)를 받는 조건 연산자를 말한다.
> 
- ? ‘ ‘ : () 형태로 쓴다
- ? 앞의 피연산자는 boolean 변수 또는 조건식이 오므로 조건 연산자라고 한다.
- 이 값이 true이면 : 앞의 피연산자가 선택되고, false이면 콜론 뒤의 피연산자가 선택된다.
- 유일하게 세 개의 피연산자를 사용한다.

![image.png](ch03%20%EC%97%B0%EC%82%B0%EC%9E%90/image%2031.png)

```java
package operator;

public class ConditionalOperationExample {
    public static void main(String[] args) {

        int score = 85;
        char grade = (score > 90) ? 'A' : ( (score > 80) ? 'B' : 'C' );

        System.out.println(score + "점은 " + grade + "등급입니다.");

    }
}

```

## 연산의 방향과 우선순위

> 복잡한 수식같은 데서 어떤 연산을 먼저 처리할지 결정하는 규칙을 말한다.
> 
- 우선순위가 정해져 있다.
- 같은 우선순위 연산자는 대부분 왼쪽에서 오른쪽으로 계산되지만 (*, /), 대입 연산자(=) 등 일부는 오른쪽에서 왼쪽으로 진행된다.

![image.png](ch03%20%EC%97%B0%EC%82%B0%EC%9E%90/image%2033.png)

>가 &보다 우선순위가 높기 때문에 먼저 처리된다.

```java
x > 0 && y < 0
```

우선순위 연산자들은 대부분 왼쪽에서 오른쪽으로 수행한다. 

![image.png](ch03%20%EC%97%B0%EC%82%B0%EC%9E%90/image%2034.png)

하지만 대입 연산자(=, +=, -=, …)는 오른쪽에서 왼쪽으로 연산을 수행한다.

```java
a = b = c = 5;
```

표를 봐도 복잡한 식에서는 헷갈리기 십상이어서 먼저 처리해야 할 연산이 있으면 ()로 묶는 것이 좋다. 

```java
int var1 = 1;
int var2 = 3;
int var3 = 2;
int result = var1 + var2 * var3; // () 미사용
int result = (var1 + var2) * var3; // () 사용
```

```java
package operator;

public class OperatorExample {
    public static void main(String[] args) {

        int a = 10;
        int b = 5;
        int c = 2;

        int result = a + b * c << 1;

        System.out.println(result);

    }
}

```
