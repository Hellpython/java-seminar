# ch06 클래스

## 6.1 객체 지향 프로그래밍

> 부품에 해당하는 객체들을 먼저 만들고, 이 객체들을 하나씩 조립해서 완성된 프로그램을 만드는 기법
> 

→ 즉 객체들의 유기적인 협력과 결합으로 파악하고자 하는 컴퓨터 프로그래밍의 패러다임을 말한다.

---

### 객체 지향 프로그래밍의 장점

- 프로그램을 보다 유연하고 변경이 용이하게 만들 수 있다는 점이다.
- 코드의 변경을 최소화하고 유지보수를 하는 데 유리하다.
- 코드의 재사용을 통해 반복적인 코드를 최소화하고, 코드를 최대한 간결하게 표현할 수 있다.
- 인간 친화적이고 직관적인 코드를 작성하기에 용이하다.

---

- 객체: 물리적으로 존재하거나 개념적인 것 중에서 다른 것과 식별 가능한 것을 말한다.
    
    → 객체 지향 프로그래밍의 가장 기본적인 단위이자 시작점
    
    → 속성 + 동작
    
    - 속성 = 상태(State) = 변수(Field): 객체가 가지고 있는 고유한 데이터(ex 이름, 나이, 색깔, 속도)
    - 동작 = 행동(Behavior) = 메서드(Method): 객체가 할 수 있는 기능(ex 웃다, 먹다, 달린다, 멈춘다)
    
    ![image.png](ch06%20%ED%81%B4%EB%9E%98%EC%8A%A4/image.png)
    
    → 즉 현실 세계를 객체(상태 + 행동)로 나누고, 이 객체들의 협력으로 프로그램을 만드는 방식이다.
    

- 객체 모델링: 현실 세계의 객체를 소프트웨어 객체로 설계하는 것을 말한다.
    
    → 현실 세계 객체의 대표 속성과 동작을 추려내어 소프트웨어 객체의 필드와 메소드로 정의하는 과정이라고 보면 된다. 그리고 이거는 먼저 하는 것이 좋다. why?
    
    - 복잡성 감소
        
        → 현실 세계는 너무 복잡하기 때문에 아 시스템에 누가 등장하는지 명확해진다.
        
    - 각 각체의 역할과 책임이 분리된다.
        
        → 협업의 기준이 되고 미리 설계해두지 않으면 나중에 맞지 않는 사태가 발생한다.
        
    - 유연한 대처
        
        → 시간과 비용을 아낄 수 있고, 클래스 설계가 자연스럽게 나온다.
        
    

### 객체의 상호작용

> 고립된 객체들이 가만히 있는 것이 아니라 서로 끊임없이 대화하는 과정
> 

→ 객체는 “내가 할 일”만 하고 나머지는 “다른 객체에게 요청”한다.

![image.png](ch06%20%ED%81%B4%EB%9E%98%EC%8A%A4/image%201.png)

- 객체들 사이의 상호작용 수단은 메소드이다.
    - 객체가 다른 객체의 기능을 이용할 때 이 메소드를 호출한다.
        
        ```java
        메소드(매개값1, 매개값2, …);
        ```
        
- 메소드 호출을 통해 객체들은 데이터를 서로 주고받는다.
    - 매개값: 메소드 이름과 함께 전달하고자 하는 데이터를 괄호() 안에 기술하는데 이러한 데이터를 말한다.
        - 메소드가 실행할 때 필요한 값이다.
    - 리턴값: 메소드의 실행의 결과이며, 호출한 곳으로 돌려주는 값이다.
        
        ![image.png](ch06%20%ED%81%B4%EB%9E%98%EC%8A%A4/image%202.png)
        
    - 메소드의 리턴값은 아래와 같이 호출한 곳에서 변수로 대입 받아 사용한다.
        
        ```java
        int result = add(10, 20);  // 리턴한 값을 int 변수에 저장
        ```
        

### 객체 간의 관계

> 객체들이 서로 어떤 방식으로 연결되고 어떤 책임과 의존을 가지는지를 나타내는 구조이다.
> 

→ 객체들 사이의 관계를 잘 맺어야 한다.

- 객체는 단독으로 존재할 수 있지만 대부분 다른 객체와 관계를 맺고 있다.
    - 관계의 종류 3가지
        - 집합관계
            
            → 완성품과 부품의 관계를 말한다. (ex 자동차는 엔진, 타이어, 핸들 등으로 구성)
            
            ![image.png](ch06%20%ED%81%B4%EB%9E%98%EC%8A%A4/image%203.png)
            
    
    - 사용 관계
        
        → 다른 객체의 필드를 읽고 변경하거나 메소드를 호출하는 관계 (ex 사람이 자동차에게 달린다, 멈춘다 등)
        
    - 상속 관계
        
        → 부모와 자식 관계 (ex 자동차가 기계의 특징(필드, 메소드)을 물려받는다면 기계와 자동차는 상속 관계)
        

### 객체 지향 프로그래밍의 특징

> 객체의 자율성과 협력을 통해 변화에 강한 프로그램을 만드는 방식이다.
> 

→ 이것으로 인해 복잡한 프로그램을 효율적으로 관리하고, 코드의 재사용성을 높인다.

- 캡슐화(Encapsulation)
    
    → 객체의 데이터(필드), 동작(메소드)을 하나로 묶고 실제 구현 내용을 외부에 감추는 것을 말한다.
    
    - 이로 인해 외부 객체는 객체 내부의 구조를 알지 못하며 객체가 노출해서 제공하는 필드와 메소드만 이용할 수 있다.
        - 잘못된 상태 변경 방지
        - 객체 스스로 무결성 유지
        - 객체 간 결합도 감소
        
        → 객체를 ‘자율적인 존재’로 만든다.
        
        ![image.png](ch06%20%ED%81%B4%EB%9E%98%EC%8A%A4/image%204.png)
        
    - 필드와 메소드를 캡슐화하여 보호하는 이유
        
        → 외부의 잘못된 사용으로 인해 객체가 손상되지 않도록 한다.
        
    - 캡슐화 된 멤버를 노출시킬 것인지 숨길 것인지를 결정하기 위해 접근 제한자를 사용한다.
        - 접근 제한자: 클래스, 변수, 메서드의 접근(사용) 범위를 지정하여 외부에서의 접근을 통제하는 키워드이다. (ex private, defalut, protected, public)
        
        ![image.png](ch06%20%ED%81%B4%EB%9E%98%EC%8A%A4/image%205.png)
        
- 상속(Inheritance)
    
    → 이미 잘 만들어진 부모 클래스이 속성과 기능을 자식 클래스가 그대로 물려받는 것을 말한다.
    
    When?
    
    → 개념적으로 같은 종류일 때, 공통 특성을 자연스럽게 공유할 떄
    
    ![image.png](ch06%20%ED%81%B4%EB%9E%98%EC%8A%A4/image%206.png)
    
    - 코드의 재사용성을 높여준다.
        
        → 부모 객체의 필드와 메소드를 자식이 그대로 사용할 수 있어 자식 객체에서 중복 코딩을 하지 않아도 된다.
        
    - 유지 보수 시간을 최소화시켜 준다.
        
        → 부모 객체의 필드와 메소드를 수정하면 모든 자식 객체들은 수정된 필드와 메소드를 사용할 수 있다.
        
    - 주의사항
        - 재사용 목저이면 실패 확률 높다.
        - 결합도가 가장 강함
- 다형성(Polymorphism)
    
    → 사용 방법은 동일하지만 실행 결과가 다양하게 나오는 성질을 말한다.
    
    - 프로그램을 구성하는 객체(부품)를 바꾸면 프로그램의 실행 성능이 다르게 나올 수 있다.
        
        ![image.png](ch06%20%ED%81%B4%EB%9E%98%EC%8A%A4/image%207.png)
        
    - 다형성을 구현하기 위해서는 자동 타입 변환과 재정의 기술이 필요하다.
    
    이로 인해
    
    - 코드 수정 없이 기능 확장
    - 조건문 제거
    - 객체 교체 가능
- 추상화(Abstraction)
    
    → 객체의 공통적인 특징을 추출하여 상위 개념(모델)을 만드는 과정을 말한다.
    
    - 불필요한 세부 사항은 제쳐두고, 사용자에게 필요한 핵심 기능만 추려내어 정의한다.
    - abstract 클래스나 interface를 통해 구현된다.
    
    이로 인해
    
    - 구현 변경에 영향 없음
    - 의존성 감소
    - 다형성의 기반

## 6.3 클래스 선언

> 객체 생성을 위한 설계도를 작성하는 작업이다.
> 

→ 어떻게 객체를 생성하고, 객체가 가져야할 데이터(필드)가 무엇이고, 객체의 동작(메소드)은 무엇인지를 정의하는 내용이 포함된다.

- 클래스 선언은 소스 파일명과 동일하게 다음과 같이 작성한다.
    - 클래스명은 첫 문자를 대문자로 하고 캐멀 스타일로 작성한다.
        - 숫자를 포함해도 되지만 첫 문자는 숫자가 될 수 없고, 특수 문자 중 $, _를 포함할 수 있다.
    
    ```java
    // 클래스 선언
    public class 클래스명 
    {
    
    }
    ```
    
    public class: 공개 클래스를 선언한다는 뜻이다.
    
    하나의 소스 파일은 복수 개의 클래스 선언을 포함할 수 있다.
    
    ```java
    package sec03;           // 패키지 선언
    
    public class SportsCar   // 클래스 선언
    {
    
    }
    
    class Tire               // 클래스 선언
    {
    
    }
    
    ```
    

**이 때 주의할 점은 소스 파일명과 동일한 클래스만 공개 클래스로 선언할 수 있다는 점이다.

그렇기에 Tire 클래스도 공개 클래스로 선언하고 싶다면 [Tire.java](http://Tire.java) 소스 파일을 별도로 생성해야 한다. 그러므로 특별한 이유가 없다면 소스 파일 하나당 클래스 하나를 선언하는 것이 좋다.

![image.png](ch06%20%ED%81%B4%EB%9E%98%EC%8A%A4/image%209.png)

공개 클래스: 어느 위치에 있든지 패키지와 상관없이 사용할 수 있는 클래스를 말한다.

## 6.4 객체 생성과 클래스 변수

- 클래스로부터 객체를 생성하려면 객체 생성 연산자인 new가 필요하다.
    
    ```java
    new 클래스()
    ```
    
    - 이 때 new 연산자 뒤에는 생성자 호출 코드가 오는데, 클래스() 형태를 가진다.
        - new연산자는
        1. 메모리 할당: 객체를 저장할 수 있는 빈 공간을 메모리에 확보한다.
        2. 생성자 호출: 클래스의 생성자 함수를 호출하여 메모리에 확보된 빈 공간을 ‘초기화’한다.(ex 객체의 초기 속성값 설정)
        3. 참조 반환: 생성된 객체가 저장된 메모리 주소(참조)를 반환한다.
        
        ![image.png](ch06%20%ED%81%B4%EB%9E%98%EC%8A%A4/image%2010.png)
        
    
    ```java
    package school;
    
    public class StudentExample {
        public static void main(String[] args) {
    
            Student s1 = new Student();
    
            System.out.println("s1 변수가 Student 객체를 참조합니다.");
    
            Student s2 = new Student();
    
            System.out.println("s2 변수가 또 다른 Student 객체를 참조합니다.");
    
        }
    
    }
    
    ```

## 6.5 클래스의 구성 멤버

> 생성자, 필드, 메소드를 클래스 구성 멤버라고 한다.
> 

![image.png](ch06%20%ED%81%B4%EB%9E%98%EC%8A%A4/image%2013.png)

- 필드: 객체의 상태나 데이터를 저장하는 변수이다.
    - 선언 형태는 변수 선언과 비슷하지만 쓰임새는 다르다.
    - ‘속성’ 또는 ‘포르퍼티’라고도 불린다.
- 생성자: 클래스로부터 객체를 생성할 때 호출되는 특별한 메소드이다.
    - 주로 new 연산자로 객체가 생성될 때 필드를 초기화하는 역할을 한다.
    - 선언 형태는 메소드와 비슷하지만, 리턴 타입이 없다.
    - 클래스 이름과 동일한 이름을 갖는 경우가 많다.
- 메소드: 객체가 수행할 수 있는 행동이나 기능을 정의하는 함수이다.
    - 객체가 수행할 동작이며, 다른 프로그램 언어에서는 함수라고 하기도 한다.
    - 객체 내부의 함수는 메소드라고 부른다.
    - 메소드는 객체와 객체간의 상호 작용을 위해 호출된다.
- 중첩 클래스: 클래스 안에 또 다른 클래스를 정의하는 것이다.
    - 특정 클래스가 오직 다른 하나의 클래스 안에서만 사용될 때, 코드의 구조를 더 논리적으로 구성하기 위해 사용된다.

### 필드 사용

> 필드값을 읽고 변경하는 것을 말한다.

## 6.7 생성자 선언과 호출

- 객체 초기화: 필드 초기화를 하거나 메소드를 호출해서 객체를 사용할 준비를 하는 것을 말한다.
    
    ```java
    클래스 변수 = new 클래스();
    							생성자 호출
    ```
    
    - new 연산자는 객체의 주소를 리턴한다.
        - 이 때 리턴된 주소는 클래스 변수에 대입되어 객체의 필드나 메소드에 접근할 때 이용된다.

### 기본 생성자

> 매개변수가 하나도 없는 생성자를 말한다.
> 
- 모든 클래스는 생성자가 존재하며, 하나 이상을 가질 수 있다.
    - 클래스에 생성자 선언이 없으면 자바 컴파일러가 눈에 보이지 않는 기본 생성자를 자동으로 추가해 준다.
        
        ```java
        public Car()
        {
           
           // 매개변수가 없는 생성자
        
        }
        ```
        

![image.png](ch06%20%ED%81%B4%EB%9E%98%EC%8A%A4/image%2020.png)

```java
Car myCar = new Car(); // 기본 생성자 호출
```

** 개발자가 생성자를 ‘단 하나라도’ 만들면, 컴파일러는 기본 생성자를 자동으로 만들어주지 않는다.

### 생성자 선언

![image.png](ch06%20%ED%81%B4%EB%9E%98%EC%8A%A4/image%2021.png)

- 생성자는 메소드와 비슷한 모양을 가지며,리턴 타입이 없다.
- 생성자의 이름은 반드시 클래스의 이름과 정확히 일치해야 한다.
- 매개변수는 new 연산자로 생성자를 호출할 때 매개값을 생성자 블록 내부로 전달한다.
    
    ```java
    Car myCar = new Car("그랜저", "검점", 300);
    ```
    
    위 코드에서 3개의 매개값을 순서대로 매개변수로 대입 받기 위해서는 아래와 같이 생성자가 선언되어야 한다.
    
    ```java
    public class Car {
    		
    		// 생성자 선언
    		Car(String model, String color, int maxSpeed)
    		{
    			...
    		}
    	
    	}
    ```
    
    ```java

    		// 생성자 선언
    		
        Car (String model, String color, int maxSpeed)
        {
    
        }

    package drive;
    
    public class CarExample {
        public static void main(String[] args) {
    ```

### 필드 초기화

> 클래스로부터 객체를 생성할 때, 그 객체가 가지는 필드에 처음으로 값을 할당해 주는 과정을 말한다.
> 
- 객체마다 동일한 값을 갖고 있다면 필드 선언 시 초기값을 대입하는 것이 좋다.
    - 반대로 객체마다 다른 값을 가져야 한다면 생성자에서 필드를 초기화하는 것이 좋다.

## 6.8 메소드 선언과 호출

> 메소드 선언: 객체의 동작을 실행 블록으로 정의를 말한다.
> 

> 메소드 호출: 실행 블록을 실제로 실행하는 것을 말한다.
> 

** 상호작용

### 메소드 선언

![image.png](ch06%20%ED%81%B4%EB%9E%98%EC%8A%A4/image%2027.png)

> 리턴 타입: 메소드가 실행한 후 호출한 곳으로 전달하는 결과값의 타입을 말한다.
> 
- 리턴값이 없는 메소드는 void로 작성해야 한다.
    
    ```java
    void powerOn() { … }              // 리턴값의 없는 메소드 선언
    double divide(int x, int y) { … } // double 타입 값을 리턴하는 메소드 선언
    ```
    
    **리턴 타입이 있는 메소드는 실행 블록 안에서 return 문으로 리턴값을 반드시 지정해야 한다.
    

> 메소드명:  메소드를 식별하고 호출하기 위해 메소드에 붙이는 이름을 말한다.
> 
- 첫 문자는 소문자로 시작하고, 캐멀 스타일로 작성한다.
- 첫 글자는 숫자일 수 없다.

```java
void run() { … }
void setSpeed(int speed) { … }
String getName() { … }
```

> 매개변수: 메소드가 외부로부터 값을 전달받기 위해 메소드 선언부에 정의하는 특별한 변수이다.
> 
- 메소드를 호출할 때 전달한 매개값을 받기 위해 사용된다.
- 전달할 매개값이 없다면 매개변수는 생략할 수 있다.

```java
double divide(int x, int y) { … }
```

> 실행블록: 메소드 호출 시 실행되는 부분을 말한다.
> 
- 중괄호’{ }’로 둘러싸인 코드의 묶음

```java
public void walk()
{                                          // 실행 블록 시작
	System.out.println("메소드가 실행됩니다.");
}                                          // 실행 블록 끝
```

```java
package carry;

public class Calculator {
    
    // 리턴값이 없는 메소드 선언
    void powerOn()
    {
        System.out.println("전원을 켭니다.");
    }
    
    // 리턴값이 없는 메소드 선언
    void powerOff()
    {
        System.out.println("전원을 끕니다.");
    }
    
    // 호출 시 두 정수 값을 전달받고, 
    // 호출한 곳으로 결과값 int를 리턴하는 메소드 선언
    int plus(int x, int y)
    {
        int result = x + y;
        return  result;      // 리턴값 지정;
    }
    
    // 호출 시 두 정수 값을 전달받고, 
    // 호출한 곳으로 결과값 double을 리턴하는 메소드 선언
    double divide(int x, int y)
    {
        double result = (double)x / (double)y;
        return result;  // 리턴값 지정;
    }
}

```

### 메소드 호출

> 메소드 호출은 즉 선언된 메소드의 코드를 실행하는 것과 동일하다.
> 
- 메소드는 객체의 동작이므로, 메소드를 호출하려면 해당 객체가 있어야 한다.
- 객체 내부: 같은 객체 내의 다른 메소드나 생성자 내부에서는 단순히 메소드명()으로 호출한다.
    
    객체 외부: 다른 객체에서는 참조 변수와 도트(.) 연산자를 이용해서 호출한다.
    
- 메소드가 매개변수를 가지고 있다면, 호출할 때 선언된 매개변수의 타입과 개수에 맞춰 적절한 매개값을 제공해야 한다.
- 메소드가 리턴하는 값이 있을 경우, 대입 연산자를 사용하여 그 값을 변수에 저장할 수 있다.
    - 이때 값을 받을 변수 타입은 메소드의 리턴 타입과 동일하거나 자동 타입 변환될 수 있어야 한다.

```java
타입 변수 = 메소드();
```

### 가변길이 매개변수

> 매개변수의 개수를 미리 정해두지 않고, 매소드 호출 시 전달되는 인수의 개수에 따라 유동적으로 처리할 있도록 하는 기능을 말한다.
> 
- 메소드를 호출할 때에는 매개변수의 개수에 맞게 매개값을 제공해야 하지만 가변길이 매개변수를 사용해서 매개변수의 개수와 상관없이 매개값을 줄 수 있다.

```java
int sum(int … values) 
{

}
```

- 메소드 호출 시 매개값을 쉼표로 구분해서 상관없이 제공할 수 있다.
    
    ```java
    int result = sum(1, 2, 3);
    int result = sum(1, 2, 3, 4, 5);
    ```
    
- 매개값들은 자동으로 배열 항목으로 변환되어 메소드에서 사용되기 때문에 메소드 호출 시 직접 배열을 매개값으로 제공해도 된다.
    
    ```java
    int[] values = { 1, 2, 3 };       int result = sum(new int[] { 1, 2, 3 });
    int result = sum(values);
    ```
    

```java
package pro;

public class Computer {

        // 가변길이 매개변수를 갖는 메소드 선언
        int sum(int ... values)
        {
            // sum 변수 선언
            int sum = 0;

            // values는 배열 타입의 변수처럼 사용
                for (int i = 0; i < values.length; i++)
                {
                    sum += values[i];

                }
            // 합산 결과를 리턴
            return sum;
        }

}

package pro;

public class ComputerExample {
    public static void main(String[] args) {

        // Computer 객체 생성
        Computer myCom = new Computer();

        // sum() 메소드 호출 시 매개값 1, 2, 3을 제공하고
        // 합산 결과를 리턴 받아 result1 변수에 대입
        int result1 = myCom.sum(1, 2, 3);
        System.out.println("result1: " + result1);

        // sum() 메소드 호출 시 매개값 1, 2, 3, 4, 5를 제공하고
        // 합산 결과를 리턴 받아 result2 변수에 대입

        int result2 = myCom.sum(1, 2, 3, 4, 5);

        System.out.println("result2: " + result2);

        // sum() 메소드 호출 시 배열을 제공하고
        // 합산 결과를 리턴 받아 result3 변수에 대입
        int[] values = { 1, 2, 3, 4, 5 };
        int result3 = myCom.sum(values);

        System.out.println("result3: " + result3);

        // sum() 메소드 호출 시 배열을 제공하고
        // 합산 결과를 리턴 받아 result4 변수에 대입

    }
}
```

### return 문

> 메소드의 실행을 강제 종료하고, 필요하다면 값을 메소드를 호출한 곳으로 되돌려주는 역할을 하는 제어문이다.
> 
- 메소드 선언에 리턴 타입이 있을 경우에는 return 문 뒤에 리턴값을 추가로 지정해야 한다.
    
    ```java
    return [리턴값];
    ```
    
- return 문 이후의 실행문은 절대 실행도지 않기 때문에 return 문 이후에 실행문을 작성하면 Unreachable code라는 컴파일 에러가 발생한다.
    
    ```java
    int plus(int x, int y) 
    {
        int result = x + y;
        return result;
        System.out.println(result); //Unreachable code
    }
    ```
    

![image.png](ch06%20%ED%81%B4%EB%9E%98%EC%8A%A4/image%2031.png)

      위 같은 경우에는 컴파일 에러가 발생하지 않는다.

```java
package gas;

public class Car {

        // 필드 선언
        int gas;

        // 리턴값이 없는 메소드로 매개값을 받아서 gas 필드값을 변경
        void setGas(int gas)
        {
            this.gas = gas;
        }

    // 리턴값이 boolean인 메소드로 gas 필드값이 0이면 false를, 0이 아니면 true를 리턴
        boolean isLeftGas()
        {
            if (gas == 0)
            {
                System. out.println("gas가 없습니다.");
                return false; // false를 리턴하고 메소드 종료
            }
            System. out.println("gas가 있습니다.");
            return true; // true를 리턴하고 메소드 종료
        }
        // 리턴값이 없는 메소드로 gas 필드값이 0이면 return 문으로 메소드를 종료
        void run()
        {
            while (true)
            {
                if (gas > 0)
                {
                    System.out.println("달립니다.(gas잔량:" + gas + ")");
                    gas -= 1;
                }
                else
                {
                    System.out.println("멈춥니다.(gas잔량:" + gas + ")");
                    return; // 메소드 종료
                }
            }
        }
}

package gas;

public class CarExample {
    public static void main(String[] args) {

        // Car 객체 생성
        Car myCar = new Car();

        // 리턴값이 없는 setGas() 메소드 호출
        myCar.setGas(5);

        // isLeftGas() 메소드를 호출해서 받은 리턴값이 true일 경우 if 블록 실행
        if(myCar.isLeftGas())
        {
            System.out.println("출발합니다.");

            // 리턴값이 없는 run() 메소드 호출
            myCar.run();
        }
        System.out.println("gas를 주입하세요.");
    }

}

```

- 메소드는 코드의 덩어리이므로 객체마다 저장한다면 중복 저장으로 인해 메모리 효율이 떨어진다.
    - 따라서 메소드 코드는 메소드 영역에 두되 공유해서 사용하고, 이때 객체 없이는 사용하지 못하도록 제한을 걸어둔 것이다.

### this 키워드

> 객체 자기 자신을 가리키는 참조 변수이다.
> 
- 객체 내부에서는 인스턴스 멤버에 접근하기 위해 this를 사용할 수 있다.
- 생성자와 메소드의 매개변수명이 인스턴스 멤버인 필드명과 동일한 경우, 인스턴스 필드임을 강조하고자 할 때 this를 주로 사용한다.

```java
package carina;

public class Car {

    //필드 선언
    String model;
    int speed;

    //생성자 선언
    Car(String model) {
        this.model = model; // 매개변수를 필드에 대입(this 생략 불가)
    }

    //메소드 선언
    void setSpeed(int speed) {
        this.speed = speed; // 매개변수를 필드에 대입(this 생략 불가)
    }

    void run()
    {
        this.setSpeed(100);
        System.out.println(this.model + "가 달립니다.(시속:" + this.speed + "km/h)");   // this 생략 가능
    }
}

package carina;

public class CarExample {
    public static void main(String[] args) {

        Car myCar = new Car("포르쉐");
        Car yourCar = new Car("벤츠");

        myCar.run();
        yourCar.run();

    }
}

```

## 6.10 정적 멤버

> 메소드 영역의 클래스에 고정적으로 위치하는 멤버를 말한다.
> 

→ 그러므로 객체 생성할 필요 없이 클래스를 통해 바로 사용이 가능하다.

![image.png](ch06%20%ED%81%B4%EB%9E%98%EC%8A%A4/image%2038.png)

### 정적 멤버 선언

- 정적 필드와 정적 메소드로 선언하려면 아래와 같이 static 키워드를 추가하면 된다.
    
    ```java
    public class 클래스 
    {
    	//정적 필드 선언
    	static 타입 필드 [= 초기값];
    	
    	//정적 메소드
    	static 리턴타입 메소드( 매개변수, … ) { … }
    }
    ```
    

- 객체마다 가지고 있을 필요성이 없는 공용적인 필드는 정적 필드로 선언하는 것이 좋다.
    - ex) Calculator 클래스에서 파이(π)는 정적 필드로 선언하는 것이 좋다.
        
        ```java
        public class Calculator 
        {
        	String color;               // 계산기별로 색깔이 다를 수 있다.
        	static double pi = 3.14159; // 계산기에서 사용하는 파이(π) 값은 동일하다.
        }
        ```
        

- 인스턴스 필드를 이용하지 않는 메소드는 정적 메소드로 선언하는 것이 좋다.
    - ex) Calulator 클래스에서 plus()메소드는 정적 메소드로 선언하는 것이 좋다.
        
        ```java
        public class Calculator 
        {
        	String color;                                       // 인스턴스 필드
        	void setColor(String color) { this.color = color; } // 인스턴스 메소드
        	static int plus(int x, int y) { return x + y; }     // 정적 메소드
        	static int minus(int x, int y) { return x – y; }    // 정적 메소드
        }
        ```
        

### 정적 멤버 사용

- 클래스가 메모리로 로딩되면 정적 멤버를 바로 사용할 수 있는데, 이때 클래스 이름과 함께 도트(.) 연산자로 접근하면 된다.
    
    ```java
    public class Calculator 
    {
    	static double pi = 3.14159;
    	static int plus(int x, int y) { … }
    	static int minus(int x, int y) { … }
    }
    ```
    

```java
double result1 = 10 * 10 * Calculator.pi;
int result2 = Calculator.plus(10, 5);
int result3 = Calculator.minus(10, 5);
```

- 정적 필드와 정적 메소드는 아래와 같이 객체 참조 변수로도 접근이 가능하다.
    
    ```java
    Calculator myCalcu = new Calculator();
    double result1 = 10 * 10 * myCalcu.pi;
    int result2 = myCalcu.plus(10, 5);
    int result3 = myCalcu.minus(10, 5);
    ```
    

- 정적 요소는 클래스 이름으로 접근하는 것이 정석이다.
    - 정적 멤버를 객체 참조 변수로 접근했을 경우
        
        ![image.png](ch06%20%ED%81%B4%EB%9E%98%EC%8A%A4/image%2039.png)

### 정적 블록

> static { … } 형태로 선언된 코드 블록을 말한다.
> 

```java
static 
{
	…
}
```

- 클래스가 메모리로 로딩될 때 자동으로 실행된다.
    - 정적 블록이 클래스 내부에 여러 객가 선언되어 있을 경우에는 선언된 순서대로 실행된다.

**생성자에서 초기화를 하지 않는 정적 필드

⇒ 정적 필드는 객체 생성 없이도 사용할 수 있기 때문에 생성자에서 초기화 작업을 하지 않는다. 생성자는 객체 생성 후에 실행되기 때문이다.

```java
package exam02;
```

### 인스턴스 멤버 사용 불가

> 정적 메소드와 정적 블록 안에서는 인스턴스 필드나 인스턴스 메소드를 사용할 수 없다.
> 
- 객체 자신의 참조인 this도 사용할 수 없다.

```java
public class ClassName 
{
	// 인스턴스 필드와 메소드 선언
	int field1;
	void method1() { … }
	
	// 정적 필드와 메소드 선언
	static int field2;
	static void method2() { … }
	
	// 정적 블록 선언
	static 
	{
		field1 = 10; (x) // 컴파일 에러
		method1(); (x)   // 컴파일 에러
		field2 = 10; (o)
		method2(); (o)
	}
	// 정적 메소드 선언
	static void Method3 
	{
		this.field1 = 10; (x) // 컴파일 에러
		this.method1(); (x)   // 컴파일 에러
		field2 = 10; (o)
		method2(); (o)
	}
}
```

- 만약 정적 메소드와 정적 블록에서 인스턴스 멤버를 사용하고 싶다면 다음과 같이 객체를 먼저 생성하고 참조 변수로 접근해야 한다.
    
    ```java
    static void Method3() 
    {
    	// 객체 생성
    	ClassName obj = new ClassName();
    	
    	// 인스턴스 멤버 사용
    	obj.field1 = 10;
    	obj.method1();
    }
    ```
    

- main() 메소드도 동일한 규칙이 적용되므로 객체 생성 없이 인스턴스 필드와 인스턴스 메소드를 main() 메소드에서 바로 사용할 수 없다.
    
    ```java
    public class Car 
    {
    	// 인스턴스 필드 선언
    	int speed;
    	
    	// 인스턴스 메소드 선언
    	void run() { … }
    	
    	// 메인 메소드 선언
    	public static void main(String[] args) 
    	{
    		speed = 60; (x) // 컴파일 에러
    		run(); (x)      // 컴파일 에러
    	}
    }
    ```
    

- 수정한 것
    
    ```java
    public static void main(String[] args) 
    {
    	// 객체 생성
    	Car myCar = new Car();
    	
    	// 인스턴스 멤버 사용
    	myCar.speed = 60;
    	myCar.run();
    }
    ```
    

```java
package exam03;

public class Car {

        // 인스턴스 필드 선언
        int speed;

        // 인스턴스 메소드 선언
        void run()
        {
            System.out.println(speed + "으로 달립니다.");
        }

        static void simulate()
        {
            // 객체 생성

            //인스턴스 멤버 사용
            myCar.speed = 200;
            myCar.run();
        }

        public static void main(String[] args) {
        // 정적 메소드 호출
        simulate();

        // 객체 생성
        Car myCar = new Car();

        // 인스턴스 멤버 사용
        myCar.speed = 60;
        myCar.run();
    }
}
```

## 6.11 final 필드와 상수

- 인스턴스 필드와 정적 필드는 언제든지 값을 변경할 수 있다.
    - 그러나 값을 변경하는 것을 막고 읽기만 허용할 때가 필요하다. 그때 final 필드와 상수를 선언한다.

### final 필드 선언

> 초기값이 저장되면 이것이 최종적인 값이 되어 프로그램 실행 도중에 수정할 수 없게 된다.
> 

```java
final 타입 필드 [= 초기값];
```

- final 필드에 초기값을 주는 방법
    - 필드 선언 시에 초기값 대입
        - ex) 고정된 값일 경우
    - 생성자에서 초기값 대입
        - ex) 복잡한 초기화 코드가 필요하거나 객체 생성 시에 외부에서 전달된 값으로 초기화 해야 하는 경우

```java
package exam01;

public class Korean {

        // 인스턴스 final 필드 선언
        final String nation = "대한민국";
        final String ssn;

        // 인스턴스 필드 선언
        String name;

         // 생성자 선언
        public Korean(String ssn, String name)
        {
            this.ssn = ssn;
            this.name = name;
        }
}

package exam01;

public class KoreanExample {
    public static void main(String[] args) {

        // 객체 생성 시 주민등록번호와 이름 전달
        Korean k1 = new Korean("123456-1234567, "양자바");

        // 필드값 읽기
        System.out.println(k1.nation);
        System.out.println(k1.ssn);
        System.out.println(k1.name);

        // Final 필드는 값을 변경할 수 없음
        // k1.nation = "USA";
        // k1.ssn = "123-12-1234";

        // 비 final 필드는 값 변경 가능
        k1.name = "김자바";

    }
}

```

### 상수 선언

> 프로그램에서 변하지 않는 고정된 값을 이름으로 정의하는 것을 말한다.
> 

→ ex) 원주율 파이나 지구의 무게 및 둘레 등

- 상수는 객체마다 저장할 필요가 없고, 여러 개의 값을 가져도 안되기 때문에 static이면서 final인 특성을 가져야 한다.
    
    ```java
    static final 타입 상수 [= 초기값];
    ```
    
- 복잡한 초기화가 필요한 경우에는 정적 블록에서 초기화할 수도 있다.
    
    ```java
    static final 타입 상수;
    static 
    {
    	상수 = 초기값;
    }
    ```
    
- 상수 이름은 모두 대문자로 작성하는 것이 관례이며, 서로 다른 단어가 혼합된 이름이라면 언더바(_)로 단어들을 연결한다.
    
    ```java
    static final double PI = 3.14159;
    static final double EARTH_SURFACE_AREA = 5.147185403641517E8;
    ```
    
- 상수는 정적 필드이므로 클래스로 접근해서 읽을 수 있다.
    
    ```java
    클래스명.상수
    ```
    

```java
package exam02;

public class Earth {

        // 상수 선언 및 초기화
        static final double EARTH_RADIUS = 6400;

        // 상수 선언
        static final double EARTH_SURFACE_AREA;

        // 정적 블록에서 상수 초기화
        static
        {
            EARTH_SURFACE_AREA = 4 * Math.PI * EARTH_RADIUS * EARTH_RADIUS;
        }
}

package exam02;

public class EarthMain {
    public static void main(String[] args) {

        //상수 읽기
        System.out.println("지구의 반지름: " + Earth.EARTH_RADIUS + "km");
        System.out.println("지구의 표면적: " + Earth.EARTH_SURFACE_AREA + "km^2");

    }
}

```

## 6.12 패키지

> 단순히 디렉토리만이 아니라 클래스의 일부분이며, 클래스를 식별하는 용도로 사용된다.
> 
- 주로 개발 회사의 도메인 이름의 역순으로 만든다.
    - ex) [mycompany.com](http://mycompany.com) 회사의 패키지는 com.mycompany가 된다.
    
    ![image.png](ch06%20%ED%81%B4%EB%9E%98%EC%8A%A4/image%2045.png)
    
- 상위 패키지와 하위 패키지를 도트(.)로 구분한다.
    - 도트는 물리적으로 하위 디렉토리임을 뜻한다.
- 클래스를 식별하는 용도로 사용되기 때문에 클래스의 전체 이름에 포함된다.
    - ex) Car 클래스가 com.mycompany 패키지에 속해 있다면 Car 클래스의 전체 이름은 com.mycompany.Car가 된다.
- 패키지에 속한 바이트코드 파일(~.class)은 따로 떼어내어 다른 디렉토리로 이동할 수 없다.

![image.png](ch06%20%ED%81%B4%EB%9E%98%EC%8A%A4/image%2046.png)

### 패키지 선언

> package 키워드와 함께 패키지 이름을 기술한 것으로, 항상 소스 파일 최상단에 위치해야 한다.
> 

```java
package 상위패키지.하위패키지;

public class 클래스명 { … }
```

- 패키지 이름은 모두 소문자로 작성하고, 패키지 이름이 서로 중복되지 않도록 회사 도메인 이름의 역순으로 작성하고, 마지막에는 프로젝트 이름을 붙여주는 것이 일반적이다.
    
    ```java
    com.samsung.projectname
    com.lg.projectname
    org.apache.projectname
    ```
    

### import 문

> 다른 패키지에 정의된 클래스나 인터페이스 등을 현재 파일에서 사용할 떄, 해당 패키지 이름을 생략하고 짧은 이름만으로 참조할 수 있도록 해주는 선언을 말한다.
> 

![image.png](ch06%20%ED%81%B4%EB%9E%98%EC%8A%A4/image%2047.png)

- import 문이 작성되는 위치는 패키지 선언과 클래스 선언 사이다.
    - import 키워드 뒤에는 사용하고자 하는 클래스의 전체 이름을 기술한다.
        - 동일한 패키지에 포함된 다수의 클래스를 사용해야 한다면 클래스 이름을 생략하고 *를 사용할 수 있다.

        ```java
        import com.hankook.*;
        ```

- 하위 패키지를 포함하지 않는다.
    - 포함하고 싶다면 두 개의 import 문이 필요하다.
    
    ```java
    import com.hankook.*;
    import com.hankook.project.*;
    ```

> 

→ 왜? 중요한 필드와 메소드가 외부로 노출되지 않도록 해 객체의 무결성(결점이 없는 성질)을 유지하기 위해서이다.

- 접근제한자는 총 4가지 있다.

### 클래스의 접근 제한

- 클래스를 어디에서나 사용할 수 있는 것은 아니며, 클래스는 public과 default 접근 제한을 가질 수 있다.
    
    ```java
    [ public ] class 클래스 { … }
    ```
    
- public 접근 제한자를 생략했다면 클래스는 default 접근 제한을 가진다.
    - 다른 패키지에서는 사용할 수 없게 된다.
    
    ![image.png](ch06%20%ED%81%B4%EB%9E%98%EC%8A%A4/image%2052.png)
    
- public 접근 제한자를 붙였다면 클래스는 public 접근 제한을 가진다.
    - 다른 패키지에서도 사용 가능하다.
    
    ![image.png](ch06%20%ED%81%B4%EB%9E%98%EC%8A%A4/image%2053.png)
    

### 생성자의 접근 제한

> 접근 제한자를 붙여서 해당 클래스의 객체를 생성할 수 있는 범위를 제어하는 것을 말한다.
> 
- 생성자가 어떤 접근 제한을 갖느냐에 따라 호출 가능 여부가 정해진다.

```java
public class ClassName {

//생성자 선언
```

> 
- 어떤 접근 제한을 갖느냐에 따라 호출 여부가 결정된다.

## 6.14 Getter와 Setter

> Setter: 외부로부터 값을 전달 받아 private 필드의 값을 설정하거나 수정한다.
> 

→ 직접적인 외부에서의 필드 접근 막고 대신 메소드를 통해 필드에 접근하는데 이는 메소드는 데이터를 검증해서 유효한 값만 필드에 저장할 수 있어서이다.

- 메소드 이름: set + 필드이름(첫 글자 대문자)

```java
private double speed;

public void setSpeed(double speed) 
{
	if(speed < 0) 
	{
		this.speed = 0;      // 매개값이 음수일 경우 speed 필드에 0으로 저장하고, 메소드 실행 종료
		return;
	} 
	else 
	{
		this.speed = speed;
	}
}
```

> Getter: private으로 선언된 필드의 값을 읽어서 외부로 반환한다.
> 

→ 필드값이 객체 외부에서 사용하기 부적절한 경우, 메소드로 적절한 값으로 변환해서 리턴할 수 있기 때문이다.

- 메소드 이름: get + 필드이름(첫 글자 대문자)

```java
private double speed; //speed의 단위는 마일

public double getSpeed() 
{
	double km = speed*1.6;  // 필드값인 마일을 km 단위로 환산 후 외부로 리턴
	return km;
}
```

![image.png](ch06%20%ED%81%B4%EB%9E%98%EC%8A%A4/image%2056.png)

- boolean일 경우 Getter는 get으로 시작하지 않고 is로 시작하는 것이 관례이다.
    
    ![image.png](ch06%20%ED%81%B4%EB%9E%98%EC%8A%A4/image%2057.png)
    

```java
package sec14;

public class Car {

        // 필드 선언
        private int speed;
        private boolean stop;

        // speed 필드의 Getter/Setter 선언
        public int getSpeed()
        {
            return speed;
        }

        public void setSpeed(int speed)
        {
            if(speed < 0)
            {
                this.speed = 0;
                return;
            }
            else
            {
                this.speed = speed;
            }
        }

        // stop 필드의 Getter/Setter 선언
        public boolean isStop()
        {
            return stop;
        }

        public void setStop(boolean stop)
        {
            this.stop = stop;
            if(stop == true) this.speed = 0;
        }
```

## 6.15 싱글톤 패턴

> 특정 클래스의 인스턴스(객체)가 전체 시스템에서 오직 하나만 생성해서 사용하는 소프트웨어 디자인 패턴 중 하나이다.
> 
- 생성자를 private 접근 제한해서 외부에서 new 연산자로 생성자를 호출할 수 없도록 하는 것이 핵심이다.

```java
private 클래스() {}
```

- 외부에서 마음대로 객체를 생성하는 것이 불가능하다.
    
    ```java
    public class 클래스 {
    
    	// private 접근 권한을 갖는 정적 필드 선언과 초기화
    	private static 클래스 singleton = new 클래스();   // 1번
    
    	// private 접근 권한을 갖는 생성자 선언
    	private 클래스() {}
    	
    	// public 접근 권한을 갖는 정적 메소드 선언
    	public static 클래스 getInstance() {.           // 2번
    		return singleton;
    
    	}
    }
    ```
    
    - 1번에서 자신의 타입으로 정적 필드를 선언하고 미리 객체를 생성해서 초기화시킨다.
        - private 접근 제한자를 붙여 외부에서 정적 필드값을 변경하지 못하도록 막는다.
    - 2번에서는 정적 필드값을 리턴하는 getInstance() 정적 메소드를 public으로 선언하였다.

- 외부에서 객체를 얻는 유일한 방법
    - getInstance() 메소드 호출
        - 리턴하는 객체는 정적 필드가 참조하는 싱글톤 객체이다.
        
        ```java
        클래스 변수1 = 클래스.getInstance();
        클래스 변수2 = 클래스.getInstance();
        ```
        
        ![image.png](ch06%20%ED%81%B4%EB%9E%98%EC%8A%A4/image%2059.png)
        

```java
package sec15;

public class Singleton {

        // private 접근 권한을 갖는 정적 필드 선언과 초기화
        private static Singleton singleton = new Singleton();

        // private 접근 권한을 갖는 생성자 선언
        private Singleton()
        {

        }

        // public 접근 권한을 갖는 정적 메소드 선언
        static Singleton getInstance()
        {
            return singleton;
        }

}

package sec15;

public class SingletonMain {
    public static void main(String[] args) {

        /*
        Singleton obj1 = new Singleton(); // 컴파일 에러
        Singleton obj2 = new Singleton(); // 컴파일 에러
        */

        // 정적 메소드를 호출해서 싱글톤 객체 얻음
        Singleton obj1 = Singleton. getInstance();
        Singleton obj2 = Singleton. getInstance();

        // 동일한 객체를 참조하는지 확인
        if (obj1 == obj2)
        {
            System.out.println("같은 Singleton 객체입니다.");
        }
        else
        {
            System.out.println("다른 Singleton 객체입니다.");
        }

    }
}
```
