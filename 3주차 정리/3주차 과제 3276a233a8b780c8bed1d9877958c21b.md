## 데이터 타입

복사를 이해하기에 앞서서 데이터 타입에 대해 알아둘 필요가 있다.

### 기본 타입

→ 메모리에 직접 값을 저장하며, 크기가 고정되어 있다.

- 변수를 선언하면 그 실제 값이 저장된다.
    
    ```java
    int age = 20;
    double average = 99.9;
    
    int myAge = age;
    myAge = 24;
    
    System.out.println("age: " + age);      // 20
    System.out.println("myAge: " + myAge);  // 24 
    
    // myAge = age 라고 할 때, age의 값 20이 복사되어 myAge에 들어간다. 
    // age와 myAge는 완전히 별개이므로 myAge를 바꿔도 age는 아무런 영향을 받지 않는다.
    
    ```
    
- myAge = age 라고 할 때, age의 값 20이 복사되어 myAge에 들어간다. 
age와 myAge는 완전히 별개이므로 myAge를 바꿔도 age는 아무런 영향을 받지 않는다.

### 참조 타입

→ 객체가 메모리에 저장된 주소를 저장합니다.

- 변수를 선언하면 그 실제 값이 아닌 데이터가 저장된 메모리의 주소(참조)가 저장된다.
    
    ```java
    public class House 
    {
    		public int rent;
    }
    
    House myHouse = new House();
    myHouse.rent = 37;
    
    House futureHouse = new House();
    futureHouse.rent = 50;
    
    System.out.println(myHouse);       // House@5f184fc6
    System.out.println(futureHouse);   // House@3feba861
    
    System.out.println(myHouse.rent);      // 37
    System.out.println(futureHouse.rent);  // 50
    
    myHouse = futureHouse;
    
    System.out.println(myHouse);       // House@5f184fc6
    System.out.println(futureHouse);   // House@5f184fc6
    
    System.out.println(myHouse.rent);      // 50
    System.out.println(futureHouse.rent);  // 50
    	
    ```

## 깊은 복사

> 객체를 복사할 때, 바깥쪽과 내부에 포함된 모든 하위 객체들까지 전부 새로 만들어서 복사하는 방식이다.
> 

→ 즉 ‘실제 값’을 새로운 메모리 공간에 복사하는 것을 의미한다.

- 여기서 말하는 실제 값은 기본타입일 경우이고 참조타입이면 실제 값을 복사하는 것이 아니라, 그 참조가 가리키는 객체 자체를 새로 만들고 새로 만든 객체의 주소를 할당하는 것이다.
- 원본과 복사본은 완전히 서로 독립되어 전혀 영향을 주지 않는 별도의 존재가 된다.
- 복사 생성자를 통해서 깊은 복사가 가능하다.
    - 복사 생성자: 기존의 다른 객체를 복사하여 새로운 객체를 만드는 생성자
        - 아래 예시에서는 ‘other’라는 객체를 받아서 새로운 객체를 만들었다.
            
            ```java
            package week3;
            
            public class Character {
            
                private String name;
                private int level;
                private int hp;
                private int maxHp;
            
                public Character(String name, int level, int hp, int maxHp)
                {
                    this.name = name;
                    this.level = level;
                    this.hp = maxHp;
                    this.maxHp = maxHp;
                }
            		// 복사 생성자
                public Character(Character other)
                {
                    this.name = other.name;
                    this.level = other.level;
                    this.hp = other.maxHp;
                    this.maxHp = other.maxHp;
                }
            ```
            
    - 명시적이고 직관적이다.
    - 다른 복사 메서드보다 자유롭다.
    - 복잡한 예외 처리 없이도 객체 복사가 가능하다.
    
    ```java
    package week3;
    
    public class Character {
    
        private String name;
        private int level;
        private int hp;
        private int maxHp;
    
        public Character(String name, int level, int hp, int maxHp)
        {
            this.name = name;
            this.level = level;
            this.hp = maxHp;
            this.maxHp = maxHp;
        }
    
        public Character(Character other)
        {
            this.name = other.name;

            this.hp = other.maxHp;
            this.maxHp = other.maxHp;
        }
    
        public void printInfo()
        {
            System.out.println("이름: " + name + ", 레벨: " + level + ", 체력: " + hp + "/" + maxHp);
        }
    
        public String getName()
        {
            return name;
        }
        public int getLevel()
        {
            return level;
        }
        public int getHp()
        {
            return hp;
        }
        public int getMaxHp()
        {
            return maxHp;
        }
    
        public void setName(String name)
        {
            this.name = name;
        }
        public void setLevel(int level)
        {
            this.level = level;
        }
        public void setHp(int hp)
        {
            this.hp = hp;
        }
    
    }
    
    ```
    
    ```java
    package week3;
    
    public class Main {
        public static void main(String[] args) {
    
            Character originalChar = new Character("양희찬", 10, 100, 100);

            System.out.print("원본 캐릭터: ");

            Character copiedChar = new Character(originalChar);
    
            System.out.println("\n--- 깊은 복사 직후 ---");
            System.out.print("원본 캐릭터: ");

            System.out.print("복사된 캐릭터: ");
            copiedChar.printInfo();

            copiedChar.setLevel(15);
            copiedChar.setHp(50);
    
            System.out.println("\n--- 변경 후 최종 결과 ---");
            System.out.print("원본 캐릭터: ");
            originalChar.printInfo();
            System.out.print("복사된 캐릭터: ");
            copiedChar.printInfo();
    
        }
    
    }
    
    ```

- Cloneable 인터페이스를 이용한 깊은 복사
    - 자바에서 복사를 실행하는 도구로 clone 메소드가 있다.
        
        → clone 메소드는 java.lang.Object 클래스에서 정의되어 있는데 Object 클래스는 Java의 클래스 계층의 기본 클래스이기 때문에 어느 클래스에도 clone 메소드를 상속하게 된다.
        
        ```java
        ackage human;
        
        import java.lang.Cloneable;
        
        public class Character implements Cloneable {
        
            private String name;
            private int level;
            private int hp;
            private int maxHp;
        
            public Character(String name, int level, int hp, int maxHp)
            {
                this.name = name;
                this.level = level;
                this.hp = maxHp;
                this.maxHp = maxHp;
            }
        
            @Override
            public Object clone() throws CloneNotSupportedException
            {
                return  super.clone();
            }
        ```
        
        - clone 메소드를 실행하려면 복사 대상이 되는 java.long.Cloneable 인터페이스를 구현할 필요가 있다.
            
            → 이 때 Cloneable 인터페이스를 구현하지 않은 클래스가 clone 메소드를 호출하면 예외가 발생한다.
            
            ![image.png](3%EC%A3%BC%EC%B0%A8%20%EA%B3%BC%EC%A0%9C/image%203.png)
            
        - clone 메소드에 의해 이루어지는 것은 필드의 내용을 그대로 복사한다는 점이다.
            - clone 메소드는 복사를 할 뿐이지 생성자를 호출하는 것은 아니며 인스턴스 생성 시 특수한 초기화를 필요로 하는 클래스에서는 clone 메소드 안에 처리를 하는 것이 좋다.
    
    ```java
    package human;
    
    import java.lang.Cloneable;
    
    public class Character implements Cloneable {
    
        private String name;
        private int level;
        private int hp;
        private int maxHp;
    
        public Character(String name, int level, int hp, int maxHp)
        {
            this.name = name;
            this.level = level;
            this.hp = maxHp;
            this.maxHp = maxHp;
        }
    
        @Override
        public Object clone() throws CloneNotSupportedException
        {
            return super.clone();
        }
    
        public void printInfo()
        {
            System.out.println("이름: " + name + ", 레벨: " + level + ", 체력: " + hp + "/" + maxHp);
        }
    
        public void setName(String name)
        {
            this.name = name;
        }
        public void setLevel(int level)
        {
            this.level = level;
        }
        public void setHp(int hp)
        {
            this.hp = hp;
        }
    
        public static interface Cloneable {
        }
    }
    
    ```
    

```java
package human;

public class Main {
    public static void main(String[] args) throws CloneNotSupportedException {
        Character originalChar = new Character("양희찬", 10, 100, 100);

        System.out.println("--- 복사 전 ---");
        System.out.print("원본 캐릭터: ");

        originalChar.printInfo();

        Character copiedChar = (Character) originalChar.clone();

        System.out.println("\n--- 깊은 복사 직후 ---");
        System.out.print("원본 캐릭터: ");
        originalChar.printInfo();

        System.out.print("복사된 캐릭터: ");
        copiedChar.printInfo();

        System.out.println("\n--- 복사된 캐릭터의 정보 변경 ---");

        copiedChar.setName("새로운 영웅");
        copiedChar.setLevel(15);
        copiedChar.setHp(50);

        System.out.println("\n--- 변경 후 최종 결과 ---");

        System.out.print("원본 캐릭터: ");
        originalChar.printInfo();
        System.out.print("복사된 캐릭터: ");
        copiedChar.printInfo();
    }

}
```

## 얕은 복사

> 객체를 복사할 때, 가장 바깥쪽만 새로 만들고, 그 안의 내용물은 원본의 것을 공유하는 방식이다.
> 

→ 즉 ‘주소 값’을 복사한다는 의미이다.

- 기본 타입이면 값을 그대로 복사하고, 참조타입이면 그 객체를 가리키는 주소(참조)만 복사한다.
    
    → 복사한 객체의 값을 변경한다고 기존 객체으 값이 변경되진 않지만 얕은 복사와 수정 가능한 내부 객체 이 둘이 만나면 바뀔 수도 있다.
    
    - 수정 가능한 내부 객체의 대표적인 예시로는 ‘List’가 있다.
- 객체 안에 포함된 모든 필드가 기본 타입이나 불변 객체일 때 사용하는 것이 좋다.
- 성능이 중요하고 메모리를 아낄 필요가 있어서 복사 후 하위 객체의 내용을 절대 수정하지 않을 때 사용하는 것이 좋다.
    - 얕은 복사도 ‘Cloneable/clone()’을 이용해 구현 가능하다.
        - super.clone()이 얕은 복사를 수행하는 기본 동작이다.
    
    ```java
    public class Warrior implements  Cloneable{
    
        private String name;
        private int level;
        private int hp;
        private int maxHp;
        private int attack;
        private int defense;
    
        public Warrior(String name, int level, int hp, int maxHp, int attack, int defense)
        {
            this.name = name;
            this.level = level;
            this.hp = hp;
            this.maxHp = maxHp;
            this.attack = attack;
            this.defense = defense;
        }
    
        @Override
        public Object clone() throws CloneNotSupportedException
        {
            return super.clone();
        }
    ```
    

- 복사 생성자를 이용해서도 얕은 복사가 가능하다.
    - 복사 생성자를 만들 때, 개발자가 내부의 참조 타입 필드를 ‘new’로 새로 만들지 않고, 원본 객체의 참조(주소)를 그대로 할당하면 얕은 복사가 된다.
        
        ```java
        package game;
        
        public class Warrior {
        
            private String name;
            private int level;
            private int hp;
            private int maxHp;
            private int attack;
            private int defense;
        
            public Warrior(String name, int level, int hp, int maxHp, int attack, int defense)
            {
                this.name = name;
                this.level = level;
                this.hp = hp;
                this.maxHp = maxHp;
                this.attack = attack;
                this.defense = defense;
            }
        
            public Warrior(Warrior other)
            {
                this.name = other.name;
                this.level = other.level;
                this.hp = other.hp;
                this.maxHp = other.maxHp;
                this.attack = other.attack;
                this.defense = other.defense;
            }
        
            public void printInfo()
            {
                System.out.println("이름: " + name + ", 레벨: " + level + ", 체력: " + hp + "/" + maxHp);
                System.out.println("공격력: " + attack + ", 방어력: " + defense);
            }
        
            public String getName() {
                return name;
            }
        
            public int getLevel() {
                return level;
            }
        
            public int getHp() {
                return hp;
            }
        
            public int getMaxHp() {
                return maxHp;
            }
        
            public int getAttack() {
                return attack;
            }
        
            public int getDefense() {
                return defense;
            }
        
            public void setName(String name) {
                this.name = name;
            }
        
            public void setLevel(int level) {
                this.level = level;
            }
        
            public void setHp(int hp) {
                this.hp = hp;
            }
        
            public void setMaxHp(int maxHp) {
                this.maxHp = maxHp;
            }
        
            public void setAttack(int attack) {
                this.attack = attack;
            }
        
            public void setDefense(int defense) {
                this.defense = defense;
            }
        
            public static void main(String[] args) {
                
                Warrior originalWarrior = new Warrior("양희찬", 20, 200, 200, 50, 30);
        
                System.out.println("--- 복사 전 ---");
        
                System.out.print("원본 전사: ");
                originalWarrior.printInfo();
                Warrior copiedWarrior = new Warrior(originalWarrior);
        
                System.out.println("\n--- 깊은 복사 직후 ---");
        
                System.out.print("원본 전사: ");
                originalWarrior.printInfo();
        
                System.out.print("복사된 전사: ");
                copiedWarrior.printInfo();
        
                System.out.println("\n--- 복사된 전사의 정보 변경 ---");
        
                copiedWarrior.setName("헤라클레스");
                copiedWarrior.setLevel(25);
                copiedWarrior.setHp(150);
        
                System.out.println("\n--- 변경 후 최종 결과 ---");
        
                System.out.print("원본 전사: ");
                originalWarrior.printInfo(); // 원본은 전혀 바뀌지 않음!
        
                System.out.print("복사된 전사: ");
                copiedWarrior.printInfo(); // 복사본만 바뀜
        
            }
        }
        
        ```

### ****깊은 복사 vs 얕은 복사****

앞에 내용을 정리해보면 

같은 복사지만 최상위 객체만 새로 생성하거나 모든 객체를 새로 생성한다는 차이가 존재하고 얕은 복사는 약간의 공유 개념이고 깊은 복산는 독립적인 느낌이다. 얕은 복사의 경우 원본 객체를 수정할시 복사본에도 영향이 갈 수도 있다. 그리고 상대적으로 얕은 복사가 더 빠르고 깊은 복사는 좀 느린 면이 있다.

얕은 복사는 수정 가능한 내부 객체가 없을 때 쓰는 것이 좋아 보이고, 복사본을 수정 했을 때 원본에 영향이 가지 않길 원한다면 깊은 복사를 사용하는 것이 좋아 보인다.
