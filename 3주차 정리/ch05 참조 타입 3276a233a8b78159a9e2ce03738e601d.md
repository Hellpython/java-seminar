# ch05 참조 타입

## 데이터 타입 분류

> 데이터 타입은 크게 실제 값을 저장하느냐, 아니면 값이 있는 주소를 저장하느냐에 따라 나뉜다.
> 

데이터 타입은 크게 기본 타입과 참조 타입으로 분류된다.

- 기본 타입(Primitive Type)
    
    → 정수, 실수, 문자, 논리 리터럴을 직접 저장하는 타입을 말한다.
    
    ```java
    // 기본 타입 변수
    int age = 24;
    double average = 99.9;
    ```
    
    - 스택(Stack) 영역에 실제 값이 저장된다.

![image.png](ch05%20%EC%B0%B8%EC%A1%B0%20%ED%83%80%EC%9E%85/image.png)

- 참조 타입(Reference Type)
    
    → 객체의 번지를 참조하는 타입을 말한다.
    
    - 객체: 데이터와 메소드로 구성된 덩어리를 말한다.
        
        → 객체 = 데이터(필드) + 메소드
        
    
    ```java
    // 참조 타입 변수
    String name = "삼성";
    String product = "memory";
    ```
    
    - 실제 객체는 힙(Heap) 영역에 생성되고, 변수는 그 객체가 있는 메모리 주소를 스택 영역에 저장한다.

![image.png](ch05%20%EC%B0%B8%EC%A1%B0%20%ED%83%80%EC%9E%85/image%201.png)

### 두 타입의 차이점

기본 타입으로 선언된 변수는 실제 값이 저장되지만 참조 타입으로 선언된 변수는 객체가 저장된 주소를 저장한다.

| 구분 |           기본 타입 |           참조 타입 |
| --- | --- | --- |
| 저장되는 것 |             값 자체 |   객체의 주소(참조값) |
| 메모리 위치 |               스택 |           스택 + 힙 |
| 타입 개수 |                8개 |             무제한 |

![image.png](ch05%20%EC%B0%B8%EC%A1%B0%20%ED%83%80%EC%9E%85/image%202.png)

기본 타입 변수는 Stack에 값이 저장되고, 참조 타입 변수는 Stack에 주소, 힙에 실체 객체가 저장되는 것을 보여주는 사진이다.

![image.png](ch05%20%EC%B0%B8%EC%A1%B0%20%ED%83%80%EC%9E%85/image%203.png)

## 메모리 사용 영역

자바 프로그램을 실행하면 JVM(자바 가상 머신)이 운영체제로부터 할당받은 메모리 영역을 용도에 따라 세 가지 주요 영역으로 나누어 관리한다.

- JVM: 자바 프로그램을 실행하기 위한 가상 컴퓨터이다.

### 메모리 영역(Runtime Data Area)

> JVM이 자바 프로그램을 실행하기 위해 메모리를 역할별로 나눠 관리하는 공간이다.
> 

→ 서로 성격이 다른 데이터들을 더 빠르고, 안전하고, 자동으로 관리하기 위해서 나눠서 관리한다. 

### 메소드 영역(Method Area)

> 프로그램이 시작될 때 생성되며, 모든 스레드가 공유하는 영역을 말한다.
> 

→ 바이트코드 파일을 읽은 내용이 저장되는 영역이다.

![image.png](ch05%20%EC%B0%B8%EC%A1%B0%20%ED%83%80%EC%9E%85/image%204.png)

바이트코드 파일: 사람이 쓴 자바 코드를 컴퓨터가 이해할 수 있게 중간 단계로 바꾼 파일이다.

스레드: 하나의 프로그램 안에서 동시에 여러 일을 처리하게 해주는 실행 흐름이다.

각 클래스별로

- 상수
- 정적 필드
- 메소드 코드
- 생성자 코드 등

→ 클래스의 정보를 한 번만 저장해서 모든 객체가 공유하기 위해 필요하다.

## 힙 영역(Heap Area)

> 프로그램 실행중에 만들어진 객체들이 저장되는 메모리 공간을 말한다.
> 

![image.png](ch05%20%EC%B0%B8%EC%A1%B0%20%ED%83%80%EC%9E%85/image%205.png)

- 가장 큰 메모리 비중을 차지한다.
- new 연산자로 생성된 객체와 배열이 저장된다.
- 참조 타입 변수는 이곳에 생성된 객체의 주소를 참조한다.

1. 크기가 동적으로 변한다.
2. 여러 메서드에서 공유 가능하다.
3. 생명 주기가 길 수 있다.

### 스택 영역(Stack Area)

> 메서드가 호출할 때마다 생기는 프레임이 저장되는 메모리 공간을 말한다.
> 

프레임: 메서드가 한 번 호출될 때 스택에 생성되는 실행 정보 묶음

![image.png](ch05%20%EC%B0%B8%EC%A1%B0%20%ED%83%80%EC%9E%85/image%206.png)

- 메소드 호출이 끝나면 프레임은 자동 제거된다.
    
    → 변수는 블록을 벗어나면 더 이상 접근할 수 없고, 메서드가 끝나면 프레임과 함께 스택에서 제거된다.
    
- 프레임 내부에는 로컬 변수 스택이 있다.
    
    → 여기에서 기본 타입 변수와 참조 타입 변수가 생성되고 제거된다.
    

## 참조 타입 변수의 ==, ≠ 연산

> 힙 영역의 실제 객체 주소(메모리 번지)를 비교하며, 두 변수가 동일한 객체를 가리키는지(같은 주소를 참조하는지) 확인한다.
> 

→ 즉 변수가 같은 객체를 가리키는지 조사하는 것이다.

![image.png](ch05%20%EC%B0%B8%EC%A1%B0%20%ED%83%80%EC%9E%85/image%207.png)

- ==
    
    → 두 참조 변수가 같은 메모리 주소를 가리키는지 비교한다. 같으면 true 다르면 false
    
    ```java
    refVar1 == refVar2   //결과: false
    refVar2 == refVar3   //결과: true
    ```
    
- ≠
    
    → 두 참조 변수가 다른 메모리 주소를 가리키는지 비교한다.
    
    같으면 true 다르면 false
    
    ```java
    refVar1 != refVar2    //결과: true
    refVar2 != refVar3    //결과: false
    ```
    

```java
public class ReferenceVariableCompareExample {
    public static void main(String[] args) {

        int[] arr1;   // 배열 변수 arr1 선언
        int[] arr2;   // 배열 변수 arr2 선언
        int[] arr3;   // 배열 변수 arr3 선언

        arr1 = new int[] { 1, 2, 3 }; // 배열 { 1, 2, 3 }을 생성하고 arr1 변수에 대입
        arr2 = new int[] { 1, 2, 3 }; // 배열 { 1, 2, 3 }을 생성하고 arr2 변수에 대입
        arr3 = arr2;   // 배열 변수 arr2의 값을 배열 변수 arr3에 대입

        System.out.println(arr1 == arr2); // arr1과 arr2 변수가 같은 배열을 참조하는지 검사
        System.out.println(arr2 == arr3); // arr2와 arr3 변수가 같은 배열을 참조하는지 검사

    }
}

```

![image.png](ch05%20%EC%B0%B8%EC%A1%B0%20%ED%83%80%EC%9E%85/image%208.png)

![image.png](ch05%20%EC%B0%B8%EC%A1%B0%20%ED%83%80%EC%9E%85/image%209.png)

arr1과 arr2가 힙 영역에서 다른 이유는?

바로 new 연산자로 인해서 그렇다. new 연산자는 실행될 때마다 메모리의 빈 공간을 찾아 새로운 객체를 생성하기 때문이다.

반면에 arr3는 arr2 변수의 번지가 대입되었기 때문에 두 변수는 동일한 번지를 가지며 같은 배열을 참조한다.

## null과 NullPointerException

### null

> 참조하는 객체가 없다
> 

→ 즉 변수가 스택 영역에 생성되기는 했으나, 힙 영역에 어떤 객체도 가리키지 않고 비어 있는 상태를 말한다.

![image.png](ch05%20%EC%B0%B8%EC%A1%B0%20%ED%83%80%EC%9E%85/image%2010.png)

- null은 초기값으로 사용할 수 있기 때문에 null로 초기화된 참조 변수는 스택 영역에 생성된다.

```java
refVar1 == null  //결과: false
refVar1 != null  //결과: true

refVar2 == null  //결과: true
refVar2 != null  //결과: false
```

### NullPointerException(NPE)

자바는 프로그램 실행 도중에 발생하는 오류를 예외(Exception)이라고 부른다.

> 변수가 null인 상태에서 객체의 데이터나 메소드를 사용하려고 할 때 발생하는 오류이다.
> 

→ 즉 주소가 없으니 힙 영역으로 찾아갈 길 자체가 없는 상태이다.

- 자바에서 가장 빈번하게 발생하는 에러 중 하나이다.

```java
int[] intArray = null;
intArray[0] = 10;         //NullPointerException
```

null인 상태에서 10을 저장하니까 NPE가 발생한 것이다.

```java
String str = null;
System.out.println("총 문자 수: " + str.length());     //NullPointerException
```

이번에도 null인 상태에서 메소드 호출하니까 NPE가 발생했다.

```java
public class NullPointerExceptionExample {
    public static void main(String[] args) {

        int[] intArray = null;

        intArray[0] = 10;   //NullPointerException

        String str = null;

        System.out.println("총 문자 수: " + str.length() ); //NullPointerException

    }
}

```

![image.png](ch05%20%EC%B0%B8%EC%A1%B0%20%ED%83%80%EC%9E%85/image%2011.png)

오류가 발생하면 예외 나오는 곳(NullPointerExceptionExample.java:6)에서 null인 상태의 참조 변수가 사용되고 있음을 알고 있어야 한다.

하지만 일부로 null을 대입하기도 한다.

```java
String hobby = "여행";
hobby = null;
```

![image.png](ch05%20%EC%B0%B8%EC%A1%B0%20%ED%83%80%EC%9E%85/image%2012.png)

어떤 변수에도 객체를 참조하지 않으면 해당 객체는 프로그램에서 사용할 수 없는 객체가 된다.  이러한 주소값을 잃어버려 더 이상 사용될 수 없는 쓰레기 객체를 가비지라고 한다.

> 다시 말해서 힙 영역에 생성되었지만, 스택 영역의 어떤 변수도 그 주소를 가리키고 있지 않은 객체를 말한다.
> 

→ 프로그램 코드에서도 해당 객체의 주소를 알 방법이 없으므로, 다시는 사용할 수 없는 상태이다.

이러한 객체를 쓰레기로 취급하고, 쓰레기 수집가(Garbage Collector)를 실행시켜 자동으로 제거한다.

제거하는 유일한 방법은 객체의 모든 참조를 없애는 것이다.

```java
String hobby = "여행";
hobby = "영화";
```

![image.png](ch05%20%EC%B0%B8%EC%A1%B0%20%ED%83%80%EC%9E%85/image%2013.png)

```java
package garbage;

public class GarbageObjectExample {
    public static void main(String[] args) {

        String hobby = "여행";
        hobby = null;

        String kind1 = "자동차";
        String kind2 = kind1;

        kind1 = null;

        System.out.println("kind2: " + kind2);

    }
}

```

## 문자열(String) 타입

- 문자열 타입은 String 객체로 생성한다.
    
    ```java
    String name;           // String 타입 변수 name 선언
    name = "홍길동";        // name 변수에 문자열 대입
    String hobby = "여행";  // String 타입 변수 hobby를 선언하고 문자열 대입
    ```
    
- 스택 영역에 name 이라는 변수가 만들어진다.
    
    힙 영역에 “홍길동” 이라는 문자열을 가진 String 객체가 생성되고 스택영역에 객체의 주소값이 저장된다.
    
- 스택 영역에 hobby라는 변수가 생성됨과 동시에 “여행” 객체의 주소값이 저장된다.
    
    힙 영역에 “여행” 이라는 새로운 String 객체가 생성된다.
    

![image.png](ch05%20%EC%B0%B8%EC%A1%B0%20%ED%83%80%EC%9E%85/image%2015.png)

### 문자열 비교

- 문자열 리터럴이 동일하다면 String 객체를 공유하도록 설계되어 있다.
    
    ```java
    String name1 = "홍길동";
    String name2 = "홍길동";
    
    System.out.println(name1 == name2); // true
    ```
    
    동일한 String 객체의 번지가 저장된다.
    
    ![image.png](ch05%20%EC%B0%B8%EC%A1%B0%20%ED%83%80%EC%9E%85/image%2016.png)
    
- new 연산자로 직접 String 객체를 생성하고 대입할 수도 있다.
    - new 연산자: 새로운 객체를 만드는 연산자로 객체 생성 연산자라고 한다.
        
        ```java
        String name1 = new String("홍길동");
        String name2 = new String("홍길동");
        
        System.out.println(name1 == name2); // false
        ```
        
        new 연산자로 각각 생성하였기 때문에 참조하는 주소값이 달라졌다.
        

![image.png](ch05%20%EC%B0%B8%EC%A1%B0%20%ED%83%80%EC%9E%85/image%2017.png)

이런 이유로 문자열 리터럴로 생성하는 것과 new 연산자로 생성하는 것에 따라 비교 연산자의 결과가 달라질 수 있다.

```java
String name1 = "홍길동";
String name2 = "홍길동";
String name3 = new String("홍길동");

name1 == name2 // true
name1 == name3 // false
```

동일한 문자열 리터럴로 생성된 객체를 참조하기 때문에 name1과 name2의 결과는 true가 나오지만 name3의 경우는 new 연산자로 생성했기 때문에 name1과는 다르다.

- String 객체끼리 상관없이 내부 문자열만을 비교할 경우에는 String 객체의 equals() 메소드를 사용한다.
    
    ```java
    boolean result = str1.equals(str2);  //문자열이 같은지 검사(대소문자 구분)
                    원본 문자열 비교 문자열
    boolean result != str1.equals(str2); //문자열이 다른지 검사
    ```
    

```java
package equals;

public class EqualsExample {
    public static void main(String[] args) {

        String strVar1 = "홍길동";
        String strVar2 = "홍길동";

        if (strVar1 == strVar2)
        {
            System. out.println("strVar1과 strVar2는 참조가 같음");
        }
        else
        {
            System. out.println("strVar1과 strVar2는 참조가 다름");
        }

        if (strVar1.equals(strVar2))
        {
            System. out.println("strVar1과 strVar2는 문자열이 같음");
        }

        String strVar3 = new String("홍길동");
        String strVar4 = new String("홍길동");

        if (strVar3 == strVar4)
        {
            System. out.println("strVar3과 strVar4는 참조가 같음");
        }
        else
        {
            System. out.println("strVar3과 strVar4는 참조가 다름");
        }

        if (strVar3.equals(strVar4))
        {
            System.out.println("strVar3과 strVar4는 문자열이 같음");
        }
    }
}

```

    ```java
            }
        }
    }
    
    ```

### 문자 추출

> String 객체에서 특정 위치의 문자를 가져오고 싶을 때는 charAt() 메소드를 사용한다.
> 
- charAt() 메소드는 매개값으로 주어진 인덱스의 문자를 리턴한다.
- 인덱스: 0에서부터 ‘문자열의 길이-1’까지의 번호를 말한다.

### 값 목록으로 다차원 배열 생성

```java
package array;

public class MultidimensionalArrayByValueListExample {
    public static void main(String[] args) {

        //2차원 배열 생성
        int[][] scores = {
                { 80, 90, 96 },
                { 76, 88 }
        };

        //배열의 길이
        System.out.println("1차원 배열 길이(반의 수): " + scores.length);
        System.out.println("2차원 배열 길이(첫 번째 반의 학생 수): " + scores[0].length);
        System.out.println("2차원 배열 길이(두 번째 반의 학생 수): " + scores[1].length);

        //첫 번째 반의 세 번째 학생의 점수 읽기
        System.out.println("scores[0][2]: " + scores[0][2]);

        //두 번째 반의 두 번째 학생의 점수 읽기
        System.out.println("scores[1][1]: " + scores[1][1]);

        //첫 번째 반의 평균 점수 구하기
        int class1Sum = 0;

        for (int i = 0; i < scores[0].length; i++)
        {
            class1Sum += scores[0][i];
        }

        double class1Avg = (double) class1Sum / scores[0].length;

        System.out.println("첫 번째 반의 평균 점수: " + class1Avg);

        //첫 번째 반의 평균 점수 구하기
        int class2Sum = 0;

        for (int i = 0; i < scores[1].length; i++)
        {
            class2Sum += scores[1][i];
        }

        double class2Avg = (double) class2Sum / scores[1].length;

        System.out.println("두 번째 반의 평균 점수: " + class2Avg);

        //전체 학생의 평균 점수 구하기
        int totalStudent = 0;
        int totalSum = 0;

        for (int i = 0; i < scores.length; i++)    //반의 수만큼 반복
        {                                      
            totalStudent += scores[i].length;      //반의 학생 수 합산
            
            for(int k = 0; k < scores[i].length; k++)     //해당 반의 학생 수만큼 반복
            {
                totalSum += scores[i][k]; //학생 점수 합산
            }
            }
        double totalAvg = (double) totalSum / totalStudent;
        System.out.println("전체 학생의 평균 점수: " + totalAvg);
    }
}
```

```java
            }

            {

            }
    
        }
```

## 배열 항목 반복을 위한 향상된 for 문

> 배열 및 컬렉션을 좀 더 쉽게 처리할 목적으로 모든 항목을 처음부터 끝까지 순차적으로 탐색할 때 사용하는 방법이다.

- 카운터 변수와 증감식을 사용하지 않고, 항목의 개수만큼 반복한 후 자동으로 for문을 빠져나간다.

```java
package statement;

public class AdvancedForExample {
    public static void main(String[] args) {

        for (int score : scores)
        {
             sum = sum + score;
        }
        System.out.println("점수 총합 = " + sum);
        
        double avg = (double) sum / scores.length;

        System.out.println("점수 평균 = " + avg);

}
```

## 열거(Enum) 타입

> 한정된 몇 가지의 값(데이터)만을 갖는 타입을 말한다.
> 

→ 예를 들면 계절(봄, 여름, 가을, 겨울), 요일(월~일)처럼 데이터가 고정되어 있다.

- 열거 타입을 사용하기 위해서는 먼저 열거 타입 이름으로 소스 파일(.java)을 생성하고 한정된 값을 코드로 정의해야 한다.
    - 첫 문자를 대문자로 하고 캐멀 스타일로 지어주는 것이 관례이다.
    
    ```java
    Week.java
    MemberGrade.java
    ProductKind.java
    ```
    
- enum 을 사용한다.
    
    ```java
    public enum Week 
    {
    
        MONDAY,
        TUESDAY,
        WEDNESDAY,
        THURSDAY,
        FRIDAY,
        SATURDAY,
        SUNDAY
        
    }
    
    ```
    

- 열거 상수는 열거 타입으로 사용할 수 있는 한정된 값을 말한다.
    - 관례적으로 알파벳으로 정의하며 모두 대문자로 작성한다.
    - 여러 단어로 구성될 경우에는 단어와 단어 사이를 언더바(스네이크 케이스)로 연결하는 것이 관례이다.
    
    ```java
    public enum LoginResult 
    {
    
    				LOGIN_SUCCESS,
    				LOGIN_FAILED
    
    }
    ```
    

- 열거 타입도 하나의 데이터 타입이므로 변수를 선언하고 사용해야 한다.
    
    ```java
    Week today;
    Week reservationDay;
    ```
    

- 열거 타입 변수에는 열거 상수를 대입할 수 있는데, ‘열거타입, 열거상수’ 형태로 작성한다.
    
    ```java
    Week today = Week.SUNDAY;
    ```
    

- 열거 타입은 참조 타입이므로 변수에 null도 대입할 수 있다.
    
    ```java
    Week birthday = null;
    ```
    

- 열거 변수의 값이 특정 열거 상수인지 비교할 때는 ==와 ≠ 연산자를 사용한다.
    
    ```java
    Week today = Week.SUNDAY;
    today == Week.SUNDAY        //결과: true
    ```
