# ch02 변수와 타입

## 변수 선언

변수는 하나의 값을 저장할 수 있는 메모리 번지에 붙여진 이름을 말한다.

변수를 통해 프로그램은 메모리 번지에 값을 저장하고 읽을 수 있게 된다.

변수 선언은 프로그램에서 데이터를 저장할 메모리 공간을 미리 확보하고 이름을 붙이는 과정을 말한다.

![image.png](ch02%20%EB%B3%80%EC%88%98%EC%99%80%20%ED%83%80%EC%9E%85/image.png)

변수 선언은 오직 정수형 변수라면 정수값만 실수형 변수라면 실수값만 저장할 수 있는 특징이 있다.

```c
private int count; 
private double average;
```

변수 이름에는 네이밍 규칙이 있다.

변수 이름은 첫 번째 글자가 문자여야 하고, 중간부터는 문자, 숫자, $, _를 포함할 수 있다. 또한 첫 문자를 소문자로 시작하되 캐멀 스타일로 작성하는 것이 관례이다.

### 헝가리안 표기법(hungarian case)

```c
private int intNum;
private String strName;
```

헝가리인 개발자가 사용하던 변수명에서 유래된 변수명으로 

변수의 자료형을 변수명의 접두어로 붙이는 방식으로 간단하게 변수명만 보고 자료형을 인식할 수 있는 방식이다.

### 카멜 표기법(camelCase)

```c
private int myNumber;
private String myName;
```

단어와 단어 사이를 대문자로 구분하는 표기법이다. 

낙타의 혹을 닮아서 낙타 표기법으로 명명되었다. 

즉 단어 + 단어 일 경우 뒤어 단어의 첫 글자를 대문자로 표기는 방식이다.

### 파스칼 표기법(pascal case)

```c
private int MyNumber;
private String Myname;
```

모든 단어의 첫 글자만을 대문자로 나타내고 나머지는 소문자로 나타낸다. 카멜 표기법과 유사한 모습을 보여주지만 모든 단어의 첫 글자를 대문자로 표기한다.

### 언더바 표기법(snake_case)

```c
private int my_number;
private String my_name;
```

변수명 단어 사이에 _(언더 바)를 사용해서 구분하는 방법이다.

단어 + 단어 일 경우, 단어_단어로 표현한다.

```c
private int count; //변수 선언
count = 5; //값 대입
```

자바에서는 변수 이름 지을 때 뿐만 아니라 소스 파일명(클래스명) 지을 때도 사용한다. 이 때는 첫 글자부터 대문자로 시작하는 것이 관례이고 뒤에 단어가 붙으면 그 단어 첫 글자에도 대문자로 쓰는 것이다. 

변수 이름은 웬만하면 무엇인지 알아보기 쉽게 의미 있는 이름을 지어주는 것이 좋다. 길이는 상관없지만 한글을 포함하지 않는 것이 관례이다.

### 변수 값 대입

변수 선언을 하고 이제 값을 저장할 수 있는데, 이때 우리는 대입 연산자인 =를 사용한다. =은 수학에서 쓰는 것과 달리 우측 값을 좌측 변수에 대입하는 연산자로 사용한다. 

변수 선언의 경우 저장되는 값의 타입과 이름만 결정한 것이지, 아직 메모리에 할당된 것은 아니다. 변수에 최초로 값이 대입 될 때 그제서야 메모리에 할당된 것이고, 해당 메모리에 값이 저장된다.

### 변수 초기화

변수를 선언한 후 처음으로 값을 할당하여 변수의 메모리 공간에 유효한 데이터를 넣어주는 과정을 의미한다. 이 때의 값을 초기값이라고 한다. 아래와 같이 변수를 선언함과 동시에 대입할 수도 있다.

```c
private int score = 90;
```

잘못된 예시

```c
int value;
int result = value + 10; //컴파일 에러
```

초기화 되지 않았기 때문에 컴파일 에러가 발생한 것을 볼 수 있다.

## 정수 타입

![image.png](ch02%20%EB%B3%80%EC%88%98%EC%99%80%20%ED%83%80%EC%9E%85/image%206.png)

정수 타입은 소수점이 없는 숫자(-1, 0, 100)를 저장하는 자료형이다. 총 5개가 있고, 메모리 할당 크기와 저장되는 값에 따라 바뀐다.

저장되는 값의 허용 범위를 모두 외울 필요는 없지만 메모리 할당 크기는 알고 있는 것이 좋다.  여기서 가장 큰 값인 long타입은 수치가 큰 데이터를 다루는 프로그램에서 사용된다. 주로 은행이나 과학 분야에서 사용되는 프로그램들이다. 

byte < short < int < long 순으로 크다.

![image.png](ch02%20%EB%B3%80%EC%88%98%EC%99%80%20%ED%83%80%EC%9E%85/image%207.png)

byte, short, int long은 모두 부호 있는 정수 타입이므로 최상위 bit는 부호 bit로 사용되고 나머지 bit는 값의 범위를 결정한다.

![image.png](ch02%20%EB%B3%80%EC%88%98%EC%99%80%20%ED%83%80%EC%9E%85/image%208.png)

최상위 bit가 1인 음수의 경우 나머지 7개의 bit를 모두 1의 보수(1은 0, 0은 1)로 바꾸고 1을 더한 값에 -를 붙이면 10진수가 된다. 

![image.png](ch02%20%EB%B3%80%EC%88%98%EC%99%80%20%ED%83%80%EC%9E%85/image%209.png)

코드에서 프로그래머가 직접 입력한 값을 리터럴이라고 부르는데, 변수에 대입할 정수 리터럴은 진수에 따라 작성하는 방법이 다르다.

### 2진수

0b 또는 0B로 시작하고 0과 1로 작성

```c
int x = 0b1011; // 10진수 값 = 8 + 0 + 2 + 1 = 11
int y = 0B10100; // 10진수 값 = 16 + 0 + 4 + 0 + 0 = 20
```

### 8진수

0으로 시작하고 0~7 숫자로 작성

```c
int x = 013; // 10진수 값 = 8 + 3 = 11
int y = 0206; // 10진수 값 = 128 + 0 + 6 = 134
```

### 10진수

소수점이 없는 0~9 숫자로 작성

```c
int x = 12; // 10진수 = 12
int y = 365; // 10진수 = 365
```

16진수 

0x 또는 0X로 시작하고 0~9 숫자나 A, B, C, D, E, F, 또는 a, b, c, d, e, f로 작성

```c
int var1 = 0b1011; // 2진수 = 11
int var2 = 0206; // 8진수 = 134
int var3 = 365; // 10진수 = 365
int var4 = 0xB3; // 16진수 = 179
```

### 허용 범위 초과한 값 오류

![image.png](ch02%20%EB%B3%80%EC%88%98%EC%99%80%20%ED%83%80%EC%9E%85/image%2010.png)

![image.png](ch02%20%EB%B3%80%EC%88%98%EC%99%80%20%ED%83%80%EC%9E%85/image%2011.png)

추가로 기본적으로 컴파일러는 정수 리터럴을 int 타입 값으로 간주하기 때문에 int 타입의 허용 범위(-2,147,483,648 ~ 2,147,483,647)를 초과하는 리터럴은 뒤에 소문자 ‘I’이나 대문자 ‘L’을 붙여 long 타입 값임을 컴파일러에게 알려줘야 한다.

```c
long var1 = 1000000000000L; // 숫자 뒤에 L을 붙임으로서 long 타입 값임을 알리기
```

## 문자 타입

하나의 문자를 작은 따옴표(’)로 감싼 것을 문자 리터럴이라고 한다. 문자 리터럴은 유니코드로 변환되어 저장되는데, 유니코드는 세계 각국의 문자를 0~65535 숫자로 매핑한 국제 표준 규약이다. 

자바에서는 이러한 유니코드를 저장할 수 있도록 char 타입을 제공한다.

```c
char var1 = 'A'; // 'A' 문자와 매핑되는 숫자: 65로 대입
char var2 = '가'; // '가' 문자와 매핑되는 숫자: 44032로 대입
```

### char 타입

유니코드가 정수이므로 char 타입도 정수 타입에 속한다. 그러므로 char 변수에 작은따옴표로 감싼 문자가 아니라 유니코드 숫자를 직접 대입할 수도 있다.

```c
char c = 65; // 10진수 65와 매핑되는 문자: 'A'
char c = 0x0041; // 16진수 0x0041과 매핑되는 문자: 'A'
```

char 타입의 변수에 어떤 문자도 대입하지 않고 단순히 초기화를 할 목적으로 쓸 수도 있다.

```c
char c = ''; // 컴파일 에러
char c = ' '; // 공백 하나를 포함해서 초기화
```

작은따옴표를 연달아 두 번 사용하게 되면 컴파일 에러가 발생하지만 ‘ ‘공백(유니코드: 32) 하나를 포함하면 초기화가 가능하다.

### 부동 소수점 방식

자바는 IEEE 754 표준에 근거해서 float 타입과 double 타입의 값을 부동 소수점 방식으로 메모리 저장한다.

![image.png](ch02%20%EB%B3%80%EC%88%98%EC%99%80%20%ED%83%80%EC%9E%85/image%2014.png)

실수 타입은 기수와 지수를 저장하기 위해 전제 bit를 나누어 사용한다.

![image.png](ch02%20%EB%B3%80%EC%88%98%EC%99%80%20%ED%83%80%EC%9E%85/image%2015.png)

부호 bit는 0이면 양수, 1이면 음수가 된다.

지수는 float 타입은 8bit, double 타입은 11bit로 표현하고 나머지는 모두 기수를 표현하는데 사용된다.

float < double 순으로 크다.

### 10진수 리터럴

```c
double x = 0.25;
double y = -3.14;

double a = 5e2; // 5.0 x 100 = 500.0
double b = 0.12E-2; // 0.12 x 10^-2 = 0.0012
```

이 때 컴파일러는 실수 리터럴을 기본적으로 double 타입으로 해석하기 때문에 double 타입 변수에 대입해야 한다. 만약 float 타입에 대입하고 싶다면 리터럴 뒤에 소문자 ‘f’나 대문자 ‘F’를 붙여 컴파일러가 float 타입임을 알 수 있도록 해야 한다.

```c
float var = 3.14f;
float var = 3E6F;
```

### 유효 자릿수 확인

double 타입은 float 타입보다 약 2배의 유효 자릿수를 가지기 때문에 보다 정확한 데이터 저장이 가능하다.

## 문자열 타입

텍스트 데이터를 표현하는 자료형이다. 작은따옴표로 감싼 한 개의 문자는 char 타입이지만 큰따옴표로 감싼 여러 개의 문자들은 유니코드로 변환되지 않는다.

```c
char var1 = "A" // 컴파일 에러
String var2 = "A" 
```

문자열을 변수에 저장하고 싶다면 String 타입을 사용해야 한다.

- String
    
    문자열 그 자체를 표현하는 타입
    
    ```c
    String x = "Hello";
    ```
    
    - 불변
    - 가장 기본이자 중심
    - 문자열 리터럴, 문자열 풀(String Pool) 사용
    - 일반적인 모든 문자열 표현에 사용
    
    즉 자바의 표준 문자열 타입
    
- StringBuilder
    
    가변 문자열(문자열을 조립할 때)
    
    ```c
    StringBuilder sb = new StringBuilder("Hello");
    ```
    
    - 가변
    - 빠름
    - 단일 스레드 환경에 최적
    
    즉 반복문, 문자열 누적 작업의 정석
    
- StringBuffer
    
    StringBuilder와 기능 동일
    
    ```c
    StringBuffer sb new StringBuffer("Hello");
    ```
    
    - synchronized → 스레드 안전
    - 성능은 느림
    
    → 요즘은 거의 안 쓰지만, 개념상 존재

## 자동 타입 변환

자동으로 타입 변환이 일어나는 것을 말한다.

언제 일어나나면 값의 허용 범위가 작은 타입이 허용 범위가 큰 타입으로 대입될 때 발생한다.

![image.png](ch02%20%EB%B3%80%EC%88%98%EC%99%80%20%ED%83%80%EC%9E%85/image%2025.png)

기본 타입을 허용 범위 순으로 나열한 경우

![image.png](ch02%20%EB%B3%80%EC%88%98%EC%99%80%20%ED%83%80%EC%9E%85/image%2026.png)

자동 타입 변환이 되는 경우를 알아보자.

```c
byte byteValue = 10;
int intValue = byteValue; // 자동 타입 변환됨
```

int 타입이 byte 타입보다 허용 범위가 더 크기 때문에 자동 타입 변환이 된다.

![image.png](ch02%20%EB%B3%80%EC%88%98%EC%99%80%20%ED%83%80%EC%9E%85/image%2027.png)

byte 타입은 1byte, int 타입은 4byte 메모리 크기를 가진다.

```c
long longValue = 50000000000L;
float floatValue = longValue; // 5.0E9f로 저장됨
double doubleValue = longValue; //5.0E9로 저장됨
```

정수 타입이 실수 타입으로 대입 될 경우에도 무조건 자동 타입 변환이 된다. 왜냐하면 실수 타입은 정수 타입보다 허용 범위가 더 크기 때문이다. 

```c
char charValue = 'A';
int intValue = charValue; // 65가 저장됨
```

char 타입의 경우에도 int 타입으로 자동 변환이 된다. 이 경우에는 자동 변환되면 유니코드 값이 int 타입에 대입된다.

이러한 자동 타입 변환에도 예외가 있는데 char 타입보다 허용 범위가 작은 byte 타입은 char 타입으로 자동 변환될 수 없다. 왜냐하면 char타입의 경우는 음수를 포함 시키지 않아서 이다. 

## 강제 타입 변환

자동 타입 변환의 경우 작은 타입에서 큰 타입으로 변환은 자동으로 되지만 큰 타입에서 작은 허용 범위 타입으로 변환은 자동이 아니라 강제로 해야 한다. 

강제 타입 변환은 캐스팅 연산자로 괄호를 사용하는데 괄호 안에 들어가는 타입은 쪼개는 단위이다.

![image.png](ch02%20%EB%B3%80%EC%88%98%EC%99%80%20%ED%83%80%EC%9E%85/image%2028.png)

### int → byte

int 타입은 byte 타입보다 더 큰 허용 범위를 가진다. 따라서 강제 타입 변환을 해야 한다.

```c
int intValue = 10;
byte byteValue = (byte) intValue; // 강제 타입 변환
```

10진수 10은 2진수 1010 이므로 1byte만 있으면 저장 가능 한데

byte 타입으로 강제 타입 변환하면 앞 3byte는 삭제되고 끝 1byte만 남아 저장된다.  

![image.png](ch02%20%EB%B3%80%EC%88%98%EC%99%80%20%ED%83%80%EC%9E%85/image%2029.png)

### long → int

long 타입은 int 타입보다 더 크다. 그러므로 강제 변환 시켜야 한다. 8byte 중에 4byte는 버려지고 남은 4byte에 저장된다.

```c
long longValue = 300;
int intValue = (int) longValue; // 강제 타입 변환 후에 300이 그대로 유지
```

### int → char

int 타입은 char 타입보다 더 큰 허용 범위를 가지므로 강제 변환 시켜야 한다. 다만 char 타입의 허용 범위인 0 ~ 65535 사이의 값만 원래 값을 유지한다.

```c
double doubleValue = 3.14;
int intValue = (int) doubleValue; // intValue는 정수 부분인 3만 저장
```

### 실수 → 정수

실수 타입은 정수 타입보다 항상 더 큰 허용 범위를 가지므로 정수 타입으로 캐스팅 해서 강제 변환 시켜야 한다. 대신 이 경우에는 소수점 이하 부분은 버려지고, 정수 부분만 저장된다.

```c
double doubleValue = 3.14;
int intValue = (int) doubleValue; // intValue는 정수 부분인 3만 저장
```

## 연산식에서 자동 변환

서로 다른 데이터 타입의 피연산자가 있을 때, 더 큰 데이터 타입으로 자동으로 변환되어 연산을 수행하는 기능을 말한다.

![image.png](ch02%20%EB%B3%80%EC%88%98%EC%99%80%20%ED%83%80%EC%9E%85/image%2030.png)

쉽게 말하면 산술 연산자(+ - * / %)가 등장하면, 피연산자의 타입이 다음 순서로 자동 변환되는 것이다.

byte, char, short → int → long → float → double

byte, char, short는 연산 순간에 무조건 int로 변한다.

사진에서 왼쪽이 왜 컴파일 에러가 나냐?

byte에서 int로 자동 변환은 되지만 int에서 byte로는 안 되기 때문이다. 그래서 강제 형 변환 또는 int result로 해야 한다.

실수 연산에 대해서 알아보자 앞에서는 byte, char, short는 전부 다 연산 시 int로 바뀌었지만 실수 연산에서는 동일한 실수 타입이라면 해당 타입으로 연산된다.

```c
float result = 1.2f + 3.4f; // 컴파일: float result = 4.6f;
```

지금은 모두가 float이라서 결과도 float으로 나왔지만 피연산자 중 하나라도 double 타입이면 다른 피연산자도 double 타입으로 변환되어 연산되고, 연산 결과 또한 double 타입이 된다.

![image.png](ch02%20%EB%B3%80%EC%88%98%EC%99%80%20%ED%83%80%EC%9E%85/image%2031.png)

int 타입 경우에도 마찬가지이다. int 타입과 double 타입이 있을 경우 double 타입으로 자동 변환되고 연산을 수행한다.

![image.png](ch02%20%EB%B3%80%EC%88%98%EC%99%80%20%ED%83%80%EC%9E%85/image%2032.png)

만약 int 타입의 결과가 필요하다면 double 타입을 강제로 int 타입으로 변환하고 수행하면 된다.

```c
int intValue = 10;
double doubleValue = 5.5;
int result = intValue + (int) doubleValue; // 10 + 5 = 15
```

또 하나의 예시를 들어보자면 

```c
int x = 1;
int y = 2;
double result = x / y;
System.out.println(result); // 0.5?
```

위 코드의 경우 0.5가 아니라 0.0 이 나온다. 자바에서 정수 연산의 결과는 항상 정수가 되기 때문이다. 만약 우리가 원하는 0.5 값이 나오려면 x / y 에서 둘 중 하나 또는 둘 다 double 타입으로 변환한다면 값이 나온다.

![image.png](ch02%20%EB%B3%80%EC%88%98%EC%99%80%20%ED%83%80%EC%9E%85/image%2033.png)

![image.png](ch02%20%EB%B3%80%EC%88%98%EC%99%80%20%ED%83%80%EC%9E%85/image%2034.png)

### (+)연산자의 기능

자바에서 + 연산자는 두 가지 기능이 있다. 

- 덧셈 연산 기능
- 문자열 결합 연산 기능

```c
int value = 3 + 7; // int value = 10;
String str = "3" + 7; // 37
STring str = "3" + "7" // 37
```

연산식에서 + 연산자는 앞에서부터 순차적으로 + 연산을 수행한다. 만약 먼저 수행된 연산이 결합 연산이라면 이후 + 연산은 모두 결합 연산이 된다. 그리고 수학처럼 괄호를 이용하면 ()부터 먼저 연산을 수행한다.

```c
String str = "1" + 2 + "3" // 123
String str = "12" + 3 = 123
String str = "1" + (2 + 3); // 5
```

### 지역 변수(Loval Variable)

함수, 반복문(for, while), 조건문(if) 등 특정 블록 내에서만 선언되고 사용되는 변수이다. 해당 블록을 벗어나면 소멸한다.

클래스 내부에 선언되어 해당 클래스의 모든 메서드에서 접근 가능하며, 객체(인스턴스)가 생성될 때 만들어진다.

특정 지역(함수, 클래스) 밖에 선언되어 프로그램 전체에서 접근 가능한 변수이다.

## 키보드 입력 데이터를 변수에 저장

키보드 입력 데이터를 변수에 저장하려면 프로그래밍 언어의 표준 입출력(Standard I/O) 기능을 사용하며 주로 Scanner를 사용한다.

### Scanner

![image.png](ch02%20%EB%B3%80%EC%88%98%EC%99%80%20%ED%83%80%EC%9E%85/image%2043.png)

### scanner.nextLine()

키보드로 입력된 내용을 문자열로 읽고 좌측 String 변수에 저장할 수 있다.

![image.png](ch02%20%EB%B3%80%EC%88%98%EC%99%80%20%ED%83%80%EC%9E%85/image%2044.png)

Enter키가 입력되기 전까지 대기 상태가 되며 키가 입력되면 지금까지 입력된 모든 내용을 문자열로 읽는다.

```c
import java.util.Scanner;

public class ScannerExample {
    public static void main(String[] args)  throws Exception {

        Scanner scanner = new Scanner(System.in);
        System.out.print("x: ");

        String strX = scanner.nextLine();
        int x = Integer.parseInt(strX);
        System.out.print("y: ");

        String strY = scanner.nextLine();
        int y = Integer.parseInt(strY);
        int result = x + y;

        System.out.println("x + y: " + result);
        System.out.println();

        while(true)
        {
            System.out.print("입력 문자열: ");
            String data = scanner.nextLine();
            if (data.equals("q"))
            {
                break;
            }
            System.out.println(": " + data);
            System.out.println();
        }
        System.out.println("종료");
    }
}

```
