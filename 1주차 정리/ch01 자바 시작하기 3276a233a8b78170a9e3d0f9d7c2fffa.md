# ch01 자바 시작하기

기계어는 컴퓨터가 이해할 수 있는 언어로 우리가 일상생활에서 사용하는 언어와는 매우 다른 모습을 보여준다.

그래서 사람이 이해하기 매우 어려운 오로지 0과 1로만 이루어진 이진 코드를 사용한다.

그렇기에 우리는 사람의 언어와 기계어를 이어주는 역할을 하는 프로그래밍 언어가 필요하다.

프로그래밍 언어에는 고급 언어와 저급 언어가 있다.

고급 언어는 고급이라는 단어처럼 컴퓨터와 대화할 수 있게 만들어졌다. 하지만 바로 이해할 수는 없어서 컴파일이라는 과정을 통해서 컴퓨터가 이해할 수 있게 앞서 말했던 것처럼 0과 1로 바꿔서 컴퓨터가 사용하게 한다.

![image.png](ch01%20%EC%9E%90%EB%B0%94%20%EC%8B%9C%EC%9E%91%ED%95%98%EA%B8%B0/image.png)

고급 언어로 작성한 파일을 소스 파일이라고 부르는데 그 종류에는 대표적으로 지금 배우는 자바가 있고 C, C++, C#, 파이썬 등이 있다.

저급 언어는 단순히 기계어에 가까운 언어를 말하는데 대표적으로 어셈블리어가 저급 언어에 속한다.

자바는 1995년도에 처음 썬 마이크로시스템즈에서 발표하였는데 지금 세간에서 가장 성공한 프로그래밍 언어로 평가 받는다. 

자바는 오라클에서 라이선스를 지니고 있으며, 자바 개발 도구 즉 JDK를 배포하여 누구나 쉽게 프로그램을 개발할 수 있도록 기술적 지원을 하였다.

자바 개발과 실행의 핵심 구성 요소로서 JDK말고 또 있다.

## JDK(Java Development Kit)

자바 프로그램 개발 및 실행에 필요한 모든 도구를 제공하는 개발 키트로 보면 된다.

JRE(→JVM + 라이브러리) + 컴파일러(javac), 디버거 등 개발도구로 구성되어 있다.

자바 개발자에게 반드시 있어야 하는 것으로 설치시 JRE와 JVM이 함께 설치된다.

## JRE(Java Runtime Environment)

이것은 자바 프로그램 실행에 필요한 최소 환경을 말하며 JVM + 핵심 라이브러리(API)로 구성되어 있으며 자바 애플리케이션을 실행만 하려면 JRE만 있어도 된다.

## JVM(Java Virtual Machine)

자바 바이트코드(.class 파일)를 각 운영체제에 맞는 기계어로 변환하여 실행하는 가상 실행 환경이다. 한 번 작성하면 어디서나 실행 즉 Write Once, Run Anywhere을 가능하게 하는 핵심 기술이다.

이처럼  JDK, JRE, JVM은 자바 개발과 실행의 핵심 구성 요소로 우리가 개발을 하기 위해서는 위 세 가지가 필요하다고 볼 수 있다.

자바는 특별히 다른 프로그래밍 언어에 비해 특별한 특징이 있는데 총 네 가지가 있다.

- 모든 운영체제에서 실행 가능
- 객체 지향 프로그래밍
- 메모리 자동 정리
- 무료 라이브러리 풍부

운영체제별 JDK를 설치하는 방법은 앞에서 말했던 것처럼 JDK를 설치하면 JRE, JVM이 둘 다 설치된다.

![image.png](ch01%20%EC%9E%90%EB%B0%94%20%EC%8B%9C%EC%9E%91%ED%95%98%EA%B8%B0/image%201.png)

JDK에는 Open JDK와 Oracle JDK가 있다.

Oracle JDK는 Open JDK보다 응답성과 JVM 성능이 상대적으로 뛰어난 모습을 보여준다. 하지만 Open JDK의 성능도 나날이 향상되고 있으며, 더욱 안정화가 되었기 때문에 JDK 비용을 고려한다면 Open JDK를 사용하는 것이 유리하다고 볼 수 있다.

![image.png](ch01%20%EC%9E%90%EB%B0%94%20%EC%8B%9C%EC%9E%91%ED%95%98%EA%B8%B0/image%202.png)

위 사진은 JDK를 다운로드 할 수 있는 사이트이다.

다운로드 받을 때 JDK LTS를 사용하는 것이 좋은 선택이다. 왜나하면 장기간 기술 지원도 받을 수 있을 뿐더러 다른 버전에 비해 안정적으로 사용할 수 있어서이다.

JDK를 설치만 한다고 끝이 아니라 윈도우 유저들은 윈도우 환경 변수 설정을 해야 한다. 운영체제의 경우 프로그램들이 실행하면서 사용할 수 있는 값들을 환경 변수 이름으로 관리합니다. 그렇기 때문에 설치 후 Path 환경 변수를 수정하는 것이 좋습니다.

![image.png](ch01%20%EC%9E%90%EB%B0%94%20%EC%8B%9C%EC%9E%91%ED%95%98%EA%B8%B0/image%203.png)

편집 버튼을 클릭 한 후 새로 만들기 다음 추가된 항목에 직접 %JAVA_HOME%\bin을 입력합니다. 입력이 끝난 뒤에는 꼭 위로 이동 버튼을 클릭해서 첫 번째 항목으로 올려줍니다. 이렇게 하는 이유는 Path 환경 변수에 등록된 순서대로 명령어를 찾기 때문인데 밑에 있을 경우 다른 버전의 Java 명령어가 사용될 가능성이 있을 수 있기 때문에 이 점을 주의해야 한다. 

마지막으로 그냥 넘어가지 말고 올바르게 되었는지 확인해야 됩니다. 

![image.png](ch01%20%EC%9E%90%EB%B0%94%20%EC%8B%9C%EC%9E%91%ED%95%98%EA%B8%B0/image%204.png)

![image.png](ch01%20%EC%9E%90%EB%B0%94%20%EC%8B%9C%EC%9E%91%ED%95%98%EA%B8%B0/image%205.png)

위 두가지 명령어를 실행하는데 만약 ‘외부 명령, 실행할 수 있는 프로그램 또는 배치 파일이 아닙니다.’라는 메시지가 출력 되면 환경 변수 설정이 잘못된 것이므로 다시 Path를 확인하고 수정 작업을 해야 한다. 수정한 뒤에는 현재 명령 프롬프트 또는 파워셀을 닫고 시작해야 한다는 점 유의해야 한다.

![image.png](ch01%20%EC%9E%90%EB%B0%94%20%EC%8B%9C%EC%9E%91%ED%95%98%EA%B8%B0/image%206.png)

이제 JDK를 설치하고 환경변수 설정까지 마쳤다면 자바 언어로 작성한 소스 파일을 만들고 컴파일할 수 있습니다. 자바의 경우 소스 파일의 확장명은 .java이다. 텍스트 파일이므로 어떤 텍스트 에디터에서도 작성이 가능하다. 

![image.png](ch01%20%EC%9E%90%EB%B0%94%20%EC%8B%9C%EC%9E%91%ED%95%98%EA%B8%B0/image%207.png)

### 바이트코드 파일

자바에서 바이트코드는 컴파일 된 결과로 생성되는 코드 이므로 일종의 명령어 집합이라고 보면 된다. 운영체제와 무관한 중간 코드 형태이며 윈도우, macOS, 리눅스에서도 동일한 바이트코드가 생성된다.

![image.png](ch01%20%EC%9E%90%EB%B0%94%20%EC%8B%9C%EC%9E%91%ED%95%98%EA%B8%B0/image%208.png)

### 자바 가상 머신(JVM)

앞에서 말한 JVM을 말하며 바이트코드 파일을 특정 운영체제가 이해하는 기계어로 변역하고 실행시킨다. JVM은 운영체제에서 이해하는 기계어로 변역해야 하므로 운영체제별로 다르게 설치해야 한다.

![image.png](ch01%20%EC%9E%90%EB%B0%94%20%EC%8B%9C%EC%9E%91%ED%95%98%EA%B8%B0/image%209.png)

```java
public class Hello {
    public static void main(String[] args) {

        System.out.println("Hello, Java");

    }
}
```

이제 설치까지 끝났으니 한 번 코드를 작성해봤습니다.

간단하게 Hello, Java를 출력해보았는데 잘 나온 것을 확인할 수 있습니다.

![image.png](ch01%20%EC%9E%90%EB%B0%94%20%EC%8B%9C%EC%9E%91%ED%95%98%EA%B8%B0/image%2010.png)

이번엔 명령 프롬프트(cmd)로 방금 만든 파일을 실행시켜보니 인텔리제이랑 똑같은 결과가 나온 것을 볼 수 있습니다.

### 코드 용어 이해

```java
public class Hello {

}
```

위 코드를 클래스 선언이라고 부르며, Hello를 클래스명이라고 지칭한다.  중괄호{…}를 클래스 블록이라고 하며, 여기에는 클래스 정의 내용이 작성된다.

클래스명을 아무렇게나 하면은 안 되고 숫자로 시작 불가능하고 공백을 포함해서는 안 된다. 그리고 소스 파일명과 대소문자가 완전히 일치해야 한다.

```java
public static void main(String[] args) {

}
```

위 부분을 main() 메소드라고 부르며 마찬가지로 중괄호 부분을 메소드 블록이라고 한다. 

### 코드 주석 달기

![image.png](ch01%20%EC%9E%90%EB%B0%94%20%EC%8B%9C%EC%9E%91%ED%95%98%EA%B8%B0/image%2011.png)

주석 코드는 프로그램 실행과는 상관없이 코드에 설명을 붙인 것으로 어디서든 작성이 가능하지만 문자열(” “) 내부에서는 작성하면 안 된다는 특징이 있다. 왜냐하면 문자열 내부에서 주석 기호는 주석문이 아니라 문자열 데이터로 인식하기 때문이다.

### 실행문과 세미콜론

![image.png](ch01%20%EC%9E%90%EB%B0%94%20%EC%8B%9C%EC%9E%91%ED%95%98%EA%B8%B0/image%2012.png)

실행문은 자바에서 실제로 명령을 수행하는 코드 단위를 말한다. 자바의 경우 실행문 뒤에는 항상 세미콜론(;)을 붙인다. 그렇지 않으면 컴파일 에러가 발생해서 오류가 나는 것을 볼 수 있다.

![image.png](ch01%20%EC%9E%90%EB%B0%94%20%EC%8B%9C%EC%9E%91%ED%95%98%EA%B8%B0/image%2013.png)

하지만 모두 다 붙이는 것은 아니고 여러 줄에 걸쳐서 작성하고 맨 마지막에 세미콜론을 붙여도 된다는 점이 있다.

```java
public class Calculator {
    public static void main(String[] args) {
        int x = 10;  //변수 선언하고 10을 저장
        int y = 9;  //변수 선언하고 9를 저장
        int result = x + y;  // 변수 result를 선언하고 x + y한 값을 저장
        System.out.println(result);  //콘솔에 출력하는 메소드 호출
    }

}
```
